SQL & MySQL Complete Notes

1. DDL (Data Definition Language)

DDL commands are used to create, change and delete database and table structure.

DDL focuses only on structure, not on data.

English Sentence:
DDL commands are used to define and modify the structure of a database.

DDL Commands

Command Meaning
CREATE Database / Table create
ALTER Change structure
DROP Delete database / table permanently
TRUNCATE Delete all data, structure remains
RENAME Rename table
COMMENT Add description

2. Client – Server Database Architecture

MySQL Server stores data centrally. MySQL Workbench is a client tool.

English Sentence:
The server stores the database centrally, and MySQL Workbench acts as the client.

Component Role
Server Stores database
Client (Workbench) Access server data
IP / Host Server address

Important:
Workbench connects only to its own server unless connected to another server.

3. Database & Workbench Relationship

MySQL Workbench is a GUI tool. It cannot store data.

English Sentence:
MySQL Workbench is just a client tool to manage databases stored in MySQL Server.

7. SQL Constraints

Constraints are rules applied to table columns to control valid data.

Constraint Use
PRIMARY KEY Unique + Not Null
FOREIGN KEY Connect tables
NOT NULL Value must be given
UNIQUE No duplicate
CHECK Condition
DEFAULT Auto value
AUTO_INCREMENT Auto numbering

8. Why ID & Primary Key are Important?

  • Name can duplicate
  • Name can change
  • ID never changes

Primary key is system identification.

ID gaps are acceptable and correct.

9. Interview Winning Sentence

SQL is a relational database, so every table must have a relationship using Primary Key and Foreign Key.
