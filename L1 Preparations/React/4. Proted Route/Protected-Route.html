🔐 Protected Routes After Login Authentication

Complete Beginner-Friendly Explanation

📌

Question Meaning (English + Tamil)

English:

"Implement protected routes after login authentication"

This means: Create a system where some pages (like Dashboard) can only be accessed if the user is logged in. If the user is NOT logged in, they should be redirected to the login page.

Tamil:

"Login authentication-க்கு அப்புறம் protected routes implement பண்ணுங்கள்"

இதன் அர்த்தம்: சில pages-ஐ (உதா: Dashboard) login பண்ணினா மட்டுமே பார்க்க முடியும் என்று ஒரு system உருவாக்கணும். Login பண்ணலன்னா, login page-க்கு redirect ஆகணும்.

🏠

Real-Life Example (நிஜ உலக உதாரணம்)

🏦 Bank Analogy (வங்கி உதாரணம்) 
1. நீங்க bank-க்கு போறீங்க (Website visit)
2. Account open பண்றீங்க (Register)
3. ATM card + PIN வாங்குறீங்க (Login → Token receive)
4. ATM card insert பண்ணி PIN போடுறீங்க (Token send to server)
5. PIN correct-ஆ இருந்தா → பணம் எடுக்கலாம் (Dashboard access)
6. PIN wrong-ஆ இருந்தா → "Invalid" message (Redirect to login)
7. Card இல்லாம ATM use பண்ண முடியாது (No token = No access)
🔑 Key Card Analogy 
Office Building = Your Website
Key Card       = Token (JWT)
Reception      = Login Page
Your Office    = Dashboard (Protected Route)
Security Guard = PrivateRoute Component + verifyToken Middleware

Without key card → Security guard won't let you in!
Key card இல்லன்னா → Security guard உள்ள விடமாட்டார்!

// Database connect பண்ற async function

  try {
    // mongoose.connect() → MongoDB-க்கு connection establish பண்ணும்
    // process.env.MONGODB_URL → .env file-ல இருக்கற MongoDB URL

    // Connection successful-ஆ இருந்தா host name print பண்ணு

  } catch (error) {
    // Connection fail-ஆனா error print பண்ணி, process-ஐ stop பண்ணு
    console.log(`MongoDb Failed To Connect ${error}`);

    // process.exit(1) → App-ஐ நிறுத்து (1 = error code)
    // DB இல்லாம app run ஆக முடியாது, அதனால stop பண்றோம்
    process.exit(1)
  }

authMiddleware.js - Token Verification (டோக்கன் சரிபார்ப்பு)

authMiddleware.js
middleware/authMiddleware.js 
// JWT = JSON Web Token library
// Token create பண்ணவும், verify பண்ணவும் use பண்றோம்
import jwt from "jsonwebtoken";

// Middleware function → request-க்கும் response-க்கும் நடுவுல run ஆகும்
// Real-life: Hotel-ல room key check பண்ற security guard
export const verifyToken = (req, res, next) => {
  try {
    // Request header-ல இருந்து Authorization value எடு
    // Header format: "Bearer eyJhbGciOiJIUzI1NiIsInR..."
    const header = req.headers.authorization;

    // Header இல்லன்னா → token அனுப்பவே இல்ல → reject
    if (!header) {
      return res.status(401).json({ msg: "No token provided" });
    }

    // "Bearer eyJhbGci..." → split(" ") → ["Bearer", "eyJhbGci..."]
    // [1] → "eyJhbGci..." (actual token மட்டும் எடுக்கறோம்)
    const token = header.split(" ")[1];

    // jwt.verify() → Token-ஐ decrypt பண்ணி verify பண்ணும்
    // JWT_SECRET = token create பண்ணும்போது use பண்ண same secret key
    // Token valid-ஆ இருந்தா → decoded = {id: "...", name: "...", iat: ..., exp: ...}
    // Token invalid/expired-ஆ இருந்தா → error throw ஆகும்
    const decoded = jwt.verify(token, process.env.JWT_SECRET);

    // Decoded user info-ஐ req.user-ல store பண்ணு
    // அடுத்த middleware/controller-ல req.user use பண்ணலாம்
    req.user = decoded;

    // next() → "OK, pass! அடுத்த function-க்கு போ"
    // next() call பண்ணலன்னா request இங்கேயே நிக்கும், response போகாது
    next();

  } catch (error) {
    // Token invalid or expired → 401 Unauthorized
    return res.status(401).json({ msg: "Invalid or expired token" });
  }
};

Why Middleware?

Without Middleware:
  Request → Controller (யாரு வேணா access பண்ணலாம் 😱)

With Middleware:
  Request → verifyToken() → Controller (token இருந்தா மட்டுமே 🛡️)

// User model - database operations பண்ண

// ═══════════════════════════════════════════════
// REGISTER FUNCTION (பதிவு செய்தல்)
// ═══════════════════════════════════════════════
export const register = async (req, res) => {
  try {
    // req.body-ல இருந்து name, email, password எடு
    // Frontend form-ல user type பண்ணது இங்க வரும்
    const { name, email, password } = req.body;

    // இந்த email already database-ல இருக்கா? check பண்ணு
    // findOne() → ஒரு document தேடும், இருந்தா return பண்ணும், இல்லன்னா null
    const existingUser = await UserModel.findOne({ email });

    // Email already இருந்தா → error message அனுப்பு
    if (existingUser) {
      return res.status(400).json({ msg: "Email already exists" });
    }

    // Password-ஐ hash பண்ணு (encrypt)
    // 10 = salt rounds (எவ்வளவு கடினமா encrypt பண்ணணும்)
    // "password123" → "$2b$10$abc123xyz789..."
    // ஏன்? → Database hack ஆனாலும் actual password தெரியாது!
    const hashedPassword = await bcrypt.hash(password, 10);

    // New user-ஐ database-ல save பண்ணு
    // Hashed password-தான் save ஆகும், actual password இல்ல!
    await UserModel.create({ name, email, password: hashedPassword });

    // Success response அனுப்பு
    return res.status(200).json({ msg: "Registered successfully" });

  } catch (error) {
    // ஏதாவது error வந்தா → 500 Internal Server Error
    return res.status(500).json({ msg: "Server error" });
  }
};

// ═══════════════════════════════════════════════
// LOGIN FUNCTION (உள்நுழைதல்)
// ═══════════════════════════════════════════════
export const login = async (req, res) => {
  try {
    // Form data-ல இருந்து email, password எடு
    const { email, password } = req.body;

    // Email-ஐ வைத்து user-ஐ database-ல தேடு
    const user = await UserModel.findOne({ email });

    // User இல்லன்னா → invalid email
    if (!user) {
      return res.status(400).json({ msg: "Invalid email" });
    }

    // bcrypt.compare() → User கொடுத்த password vs DB-ல இருக்கற hashed password
    // "password123" vs "$2b$10$abc123..." → true/false
    // bcrypt internally hash பண்ணி compare பண்ணும்
    const isMatch = await bcrypt.compare(password, user.password);

    // Password match ஆகலன்னா → invalid password
    if (!isMatch) {
      return res.status(400).json({ msg: "Invalid password" });
    }

    // ✅ Email + Password correct → JWT Token உருவாக்கு
    // jwt.sign(payload, secret, options)
    // payload = token-ல என்ன data store பண்ணணும் (user id, name)
    // secret = encryption key (.env file-ல)
    // expiresIn = token எவ்வளவு நேரம் valid (1 hour)
    const token = jwt.sign(
      { id: user._id, name: user.name },  // Token-ல store ஆகும் data
      process.env.JWT_SECRET,              // Secret key
      { expiresIn: "1h" }                  // 1 மணி நேரம் valid
    );

    // Token-ஐ frontend-க்கு அனுப்பு
    return res.status(200).json({ msg: "Login successful", token });

  } catch (error) {
    return res.status(500).json({ msg: "Server error" });
  }
};

import { Navigate } from "react-router-dom";

// children → PrivateRoute-க்குள்ள இருக்கற component (Dashboard)
// <PrivateRoute><Dashboard /></PrivateRoute>
// இங்க children = <Dashboard />
const PrivateRoute = ({ children }) => {

  // localStorage-ல "token" key-ல value இருக்கா? check பண்ணு
  // Login success-ஆனப்போ token save பண்ணியிருப்போம்
  const token = localStorage.getItem("token");

  // Ternary operator:
  // token இருந்தா (truthy) → children-ஐ render பண்ணு (Dashboard காட்டு)
  // token இல்லன்னா (falsy) → /login page-க்கு redirect பண்ணு
  return token ? children : <Navigate to="/login" />;
};

export default PrivateRoute;

// axios → HTTP requests அனுப்ப (fetch-க்கு alternative, more features)
import axios from "axios";

// useNavigate → Page redirect/navigate பண்ண (programmatically)
import { useNavigate } from "react-router-dom";

const Register = () => {

  // navigate function → வேற page-க்கு போக use பண்றோம்
  // navigate("/login") → login page-க்கு போகும்

  // form state → name, email, password values store பண்ண

Dashboard.jsx - Protected Dashboard (பாதுகாப்பான Dashboard)

Dashboard.jsx
src/pages/Dashboard.jsx 
// useEffect → Component load ஆகும்போது ஒரு தடவை run ஆகும்
// useState → Data store பண்ண
import { useEffect, useState } from "react";
import axios from "axios";
import { useNavigate } from "react-router-dom";

const Dashboard = () => {
  const navigate = useNavigate();

  // Dashboard data store பண்ண (null = இன்னும் load ஆகல)
  const [data, setData] = useState(null);

  // useEffect → Component mount ஆகும்போது (first render) run ஆகும்
  // [] (empty dependency array) → ஒரே ஒரு தடவை மட்டும் run ஆகும்
  useEffect(() => {

    // localStorage-ல இருந்து token எடு
    const token = localStorage.getItem("token");

    // Token இல்லன்னா → login page-க்கு redirect
    // Double protection: PrivateRoute check + இந்த check
    if (!token) {
      navigate("/login");
      return; // function-ஐ இங்கேயே நிறுத்து
    }

    // API call பண்ற async function
    const fetchData = async () => {
      try {
        // Dashboard API-க்கு GET request + token header-ல அனுப்பு
        const res = await axios.get(
          "http://localhost:5000/api/auth/dashboard",
          {
            headers: {
              // Bearer token format → standard authentication header
              // Backend-ல verifyToken middleware இதை read பண்ணும்
              Authorization: `Bearer ${token}`,
            },
          }
        );

        // API response data-ஐ state-ல store பண்ணு
        setData(res.data);

      } catch (error) {
        // Token invalid/expired → remove token + redirect to login
        // Real-life: ATM card expire ஆனா → new card வாங்கணும்
        localStorage.removeItem("token");
        navigate("/login");
      }
    };

    // Function-ஐ call பண்ணு
    fetchData();

  }, []); // [] → only once when component mounts

  // Logout function
  const handleLogout = () => {
    // Token-ஐ remove பண்ணு (logged out state)
    // Real-life: ATM card-ஐ return பண்றது போல
    localStorage.removeItem("token");

    // Login page-க்கு redirect
    navigate("/login");
  };

  return (
    <div>
      <h2>Dashboard</h2>

      {/* data load ஆனா மட்டும் welcome message காட்டு */}
      {/* data = null → இந்த line skip ஆகும் (conditional rendering) */}
      {data && <p>Welcome, {data.currentUser.name}!</p>}

      {/* data load ஆனா → users list-ஐ loop பண்ணி காட்டு */}
      {/* .map() → array-ல ஒவ்வொரு item-க்கும் JSX return பண்ணும் */}
      {/* key={u._id} → React-க்கு unique identifier (performance optimization) */}
      {data && data.users.map((u) => (
        <div key={u._id}>{u.name} - {u.email}</div>
      ))}

      {/* Logout button */}
      <button onClick={handleLogout}>Logout</button>
    </div>
  );
};

export default Dashboard;

5. User LOGS OUT → token removed → redirected to login
6. Token EXPIRED → backend rejects → redirect to login

🔑 Key Idea: Token = Your digital key to access pages!
🛡️ PrivateRoute = Frontend security guard
🔒 verifyToken = Backend security guard (real protection!)
