What Are the Differences Between Variables Created Using var, let, and const?

The main differences between var, let, and const in JavaScript are related to scope, reassignment & redeclaration, and hoisting behavior.

Key Differences (Quick View)

Feature var let const
Scope Function / Global scoped Block scoped Block scoped
Reassignment Yes Yes No (reference cannot change)
Redeclaration Yes No No
Hoisting Hoisted & initialized as undefined Hoisted but not initialized Hoisted but not initialized
Temporal Dead Zone No Yes Yes

Detailed Explanation

1. Scope

var is function-scoped (or global). A variable declared inside a block like if or for is still accessible outside the block within the same function.

let and const are block-scoped, meaning they are only accessible within { }, which makes code more predictable and safer.

2. Reassignment

  • var and let allow reassignment.
  • const does not allow reassignment.

Note: Objects and arrays declared with const can be mutated, but the reference itself cannot change.

3. Redeclaration

  • var allows redeclaration in the same scope, which can lead to bugs.
  • let and const do not allow redeclaration and throw a SyntaxError.

4. Hoisting and Temporal Dead Zone (TDZ)

Hoisting moves declarations to the top during the compilation phase.

  • var is hoisted and initialized with undefined, so it can be accessed before declaration.
  • let and const are hoisted but not initialized and stay in the Temporal Dead Zone (TDZ) until declared. Accessing them early causes a ReferenceError.

Modern Best Practices

  • Avoid var in modern JavaScript.
  • Use const by default for values that do not change.
  • Use let only when reassignment is required (e.g., loop counters).
