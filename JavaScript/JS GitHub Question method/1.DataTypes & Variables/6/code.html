JavaScript Number Conversion & isNaN()

This page explains parseInt(), parseFloat(), Number() and how isNaN() helps avoid calculation errors.

1. String to Number Conversion

Example 1: Numeric String

let values = "123.45";

parseInt(values);   // 123
parseFloat(values); // 123.45
Number(values);     // 123.45

Explanation:

  • parseInt() ‚Üí integer part only
  • parseFloat() ‚Üí decimal allowed
  • Number() ‚Üí converts full string to number

Example 2: String with characters

let value = "100px";

parseInt(value);   // 100
parseFloat(value); // 100
Number(value);     // NaN

Why Number() fails?

Number() requires the string to be fully numeric. "100px" contains characters ‚Üí conversion fails ‚Üí NaN.

2. What is NaN?

NaN means Not a Number. It appears when a numeric conversion fails.

3. Why isNaN() is Important?

We use isNaN() to check whether a value is a valid number before using it in calculations.

Example 3: Conversion Failure Check

let value1 = Number("abc");

if (isNaN(value1)) {
  console.log("Conversion failed");
} else {
  console.log("Conversion successful");
}

Output: Conversion failed

Example 4: Safe Calculation (Real Use Case)

let inputs = "100px";
let numbers = Number(inputs);

if (isNaN(numbers)) {
  console.log("Invalid number");
} else {
  console.log(numbers + 20);
}

Output: Invalid number

Example 5: Valid Number

let input = "50";
let number = Number(input);

if (isNaN(number)) {
  console.log("Invalid");
} else {
  console.log(number + 10);
}

Output: 60

4. Quick Comparison Table

Method Behavior
parseInt() Extracts integer, ignores characters
parseFloat() Extracts decimal number
Number() Strict conversion (fails if not fully numeric)
isNaN() Checks if value is NOT a valid number

Interview Punchlines (Must Remember)

  • parseInt and parseFloat can extract numbers from strings.
  • Number() is strict and fails if string contains characters.
  • isNaN() prevents invalid calculations.
