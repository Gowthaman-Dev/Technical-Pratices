What is the Difference Between null, undefined, and undeclared in JavaScript?

Core Difference (Easy to Remember)

The difference lies in why the value is missing:

  • null ‚Üí intentional absence (set by developer)
  • undefined ‚Üí uninitialized (set by JavaScript)
  • undeclared ‚Üí variable does not exist in the current scope

1. null

  • Represents an intentional empty value
  • Must be explicitly assigned by the developer
  • Commonly used to reset or clear a variable
let a = null;

typeof a; // "object" (historical JavaScript bug)

2. undefined

  • Assigned automatically by JavaScript
  • Means a variable is declared but not assigned a value
let b;

typeof b; // "undefined"

3. Undeclared Variable

  • Variable has not been declared using var, let, or const
  • Accessing it directly causes a ReferenceError
console.log(x); // ReferenceError (if x is undeclared)

null is an intentional empty value, undefined means a variable is declared but not initialized, and an undeclared variable does not exist in the current scope and causes a ReferenceError when accessed.
