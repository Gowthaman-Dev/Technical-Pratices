Explain the Difference Between var, let, and const

Answer

var, let, and const are JavaScript keywords used to declare variables. They mainly differ in scope, reassignment/redeclaration, and hoisting behavior.

1. Scope

  • var is function-scoped (or global if declared outside a function).
  • let and const are block-scoped, meaning they are limited to { }.

2. Reassignment and Redeclaration

  • let â†’ Can be reassigned, but cannot be redeclared in the same scope.
  • const â†’ Cannot be reassigned or redeclared.

Note: For objects and arrays declared with const, their properties can still be modified.

3. Hoisting and Temporal Dead Zone (TDZ)

  • var â†’ Hoisted and initialized with undefined, so it can be accessed before declaration.
  • let and const â†’ Hoisted but not initialized and remain in the Temporal Dead Zone (TDZ) until declared. Accessing them early causes a ReferenceError.

When to Use Which

  • const (default choice): Use when the variable should not be reassigned.
  • let: Use when reassignment is required, such as loop counters.
  • var: Avoid in modern JavaScript because function scoping and hoisting can cause bugs; use only for legacy code.
