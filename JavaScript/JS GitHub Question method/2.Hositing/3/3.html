How Does Hoisting Affect Function Declarations and Function Expressions?

In JavaScript, hoisting affects function declarations and function expressions differently due to how memory is allocated during the compilation (memory creation) phase.

1. Function Declarations

Function declarations are fully hoisted, meaning both the function name and its body are moved to the top of their scope.

Because the entire function is stored in memory before execution begins, it can be called anywhere within its scope, even before it appears in the source code.

Example

sayHello();

function sayHello() {
  console.log("Hello");
}

✅ This works without error.

Explanation

During the memory creation phase, JavaScript allocates memory for the entire sayHello function, making it available before execution starts.

2. Function Expressions

Function expressions are not fully hoisted because they depend on variable assignment.

Only the variable declaration is hoisted, not the function value assigned to it.

The behavior depends on the keyword used to declare the variable.

a) Function Expression with var

sayHi();

var sayHi = function () {
  console.log("Hi");
};

❌ TypeError: sayHi is not a function

Explanation

  • var sayHi is hoisted and initialized with undefined
  • The function assignment happens during execution
  • Calling sayHi() before assignment fails because sayHi is undefined

b) Function Expression with let or const

sayHey();

let sayHey = function () {
  console.log("Hey");
};

❌ ReferenceError

Explanation

  • The variable is hoisted but remains in the Temporal Dead Zone (TDZ)
  • Accessing it before initialization causes a ReferenceError
  • The same behavior applies to const

Summary of Differences

Feature Function Declaration Function Expression
Hoisting behavior Fully hoisted (name and body) Only variable declaration is hoisted
Can be called before definition ✅ Yes ❌ No
Error type (if called early) None TypeError (var) / ReferenceError (let/const)
Syntax function fn() {} const fn = function() {}

Short Interview Answer

Function declarations are fully hoisted and can be invoked before their definition, while function expressions are not fully hoisted because only the variable declaration is hoisted, not the function assignment.
