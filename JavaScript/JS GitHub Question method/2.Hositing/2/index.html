Explain the Difference in Hoisting Between var, let, and const

In JavaScript, all variable declarations (var, let, and const) are hoisted to the top of their scope.

The key difference lies in how they are initialized and accessed before declaration, especially with respect to the Temporal Dead Zone (TDZ).

Comparison Table

1. var Hoisting

Variables declared with var are hoisted to the top of their function or global scope and are automatically initialized with undefined.

This allows the variable to be accessed before its declaration, but its value will be undefined until the assignment runs.

Example

console.log(a); // undefined
var a = 10;
console.log(a); // 10

Explanation

The declaration var a is hoisted and initialized with undefined during the memory creation phase.

2. let Hoisting

Variables declared with let are hoisted but not initialized.

They remain in the Temporal Dead Zone (TDZ) from the start of the block until the declaration line is executed.

Accessing them before declaration causes a ReferenceError.

Example

console.log(b); // ReferenceError
let b = 20;
console.log(b); // 20

3. const Hoisting

const behaves the same as let in terms of hoisting and TDZ.

It is hoisted but not initialized, and accessing it before declaration throws a ReferenceError.

Additionally

  • const must be initialized at declaration
  • const cannot be reassigned

Example

console.log(c); // ReferenceError
const c = 30;
console.log(c); // 30

Interview Summary (Short Answer)

All variables in JavaScript are hoisted, but var is initialized with undefined, while let and const remain in the Temporal Dead Zone until declared.

Best Practice

  • Use const by default
  • Use let if reassignment is needed
  • Avoid var due to function scoping and unexpected hoisting behavior
