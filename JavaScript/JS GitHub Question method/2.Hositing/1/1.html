JavaScript Hoisting (Simple Tamil)

Definition

Hoisting is a JavaScript behavior where variable and function declarations are moved to the top of their scope during the memory creation phase, before code execution.

Code Execution Phases

1. Memory Creation Phase

  • Variable declarations are stored in memory.
  • Function declarations are stored in memory.
  • Variable initialization depends on type:
    • var → undefined
    • let / const → hoisted but in Temporal Dead Zone (TDZ)
    • Function declarations → fully ready

2. Execution Phase

Code executes line by line.

Key Points to Remember

Type Hoisted? Initial Value / Behavior
var Yes undefined
let Yes TDZ (cannot access before declaration)
const Yes TDZ (cannot access before declaration)
Function Declaration Yes Fully hoisted, can call before definition
Function Expression No Depends on var/let/const it’s stored

Examples

1️⃣ var Hoisting

console.log(a); // undefined
var a = 5;
console.log(a); // 5

2️⃣ let / const Hoisting

console.log(b); // ReferenceError
let b = 10;

3️⃣ Function Declaration

greet(); // Hello
function greet() {
  console.log("Hello");
}

4️⃣ Function Expression

sayHi(); // TypeError
var sayHi = function() {
  console.log("Hi");
};
