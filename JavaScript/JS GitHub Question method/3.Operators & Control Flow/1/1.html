Difference Between == and ===

== (Double Equals – Loose Equality)

== compares only the values. JavaScript automatically converts data types before comparison.

5 == '5' → true
('5' is converted to number 5)

=== (Triple Equals – Strict Equality)

=== compares both value and data type. It does NOT perform type conversion.

5 === '5' → false
(Number ≠ String)

Which One Should We Use?

In JavaScript, === is recommended because it avoids unexpected bugs caused by automatic type conversion.

== should be used only when type conversion is intentionally needed.

⭐ One-line Interview Answer:
“== checks only value with type coercion, while === checks both value and type without type coercion.”
