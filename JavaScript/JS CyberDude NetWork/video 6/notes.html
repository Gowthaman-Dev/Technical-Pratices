3Ô∏è‚É£ Why We Need a Set

A Set is a special JavaScript object that:

  • Stores only unique values
  • Automatically removes duplicates
  • Maintains insertion order
  • Can store any data type (string, number, object, function, etc.)

4Ô∏è‚É£ Creating a Set from an Array

const dailtset = new Set(dailyscheduls)

What happens internally?

  • JavaScript reads each element from the array one by one
  • If the value already exists ‚Üí it is ignored
  • If the value is new ‚Üí it is added

Result:

  • "work" appears only once
  • All other values remain

This is why Sets are commonly used to remove duplicates from arrays.

5Ô∏è‚É£ Adding New Values to the Set

dailtset.add('play')
  • .add() inserts a new value into the Set
  • If the value already exists, nothing happens
  • "play" is added successfully because it is unique

6Ô∏è‚É£ Adding an Object to the Set

const obj = { 'meeting': ['friends'] }
dailtset.add(obj)

Important concept: Reference types

  • Objects are stored in memory by reference
  • Even if two objects look the same, they are not equal unless they share the same reference

The Set now contains:

  • Strings (primitive values)
  • One object (obj)

7Ô∏è‚É£ Checking Values Using .has()

  • .has() checks whether a value exists in the Set
  • Returns a boolean
  • true ‚Üí value exists
  • false ‚Üí value does not exist

Since "play" was added, the output is:

true

9Ô∏è‚É£ Deleting an Object ‚Äì Common Mistake

  • This creates a new object
  • JavaScript compares objects by reference, not by structure
  • The Set does not recognize this as the same object
