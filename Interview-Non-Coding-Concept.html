01
Privacy Policy
Definition
Privacy Policy என்றால் என்ன?

Privacy Policy என்பது ஒரு website அல்லது application பயனர்களின் (users) தகவல்களை எப்படி சேகரிக்கிறது, பயன்படுத்துகிறது, சேமிக்கிறது, பாதுகாக்கிறது என்பதைக் கூறும் legal document.

Example

நீ ஒரு website-ல் register பண்ணும்போது Name, Email, Phone Number கொடுக்கிறாய். அப்போது Privacy Policy-ல்:

  • என்ன data collect பண்ணுறோம்?
  • ஏன் collect பண்ணுறோம்?
  • யாருடன் share பண்ணுவோம்?
  • எவ்வளவு நாள் store பண்ணுவோம்?
  • எப்படி protect பண்ணுவோம்?

என்பதை விளக்கி இருக்கும்.

🎤
Interview Answer

"Privacy Policy is a document that explains how an application collects, uses, stores, and protects user data. It helps users understand what happens to their personal information and builds trust between the user and the company."

Simple English

"Privacy Policy tells users what data we collect and how we use and protect it."

MERN Project Example நீ ஒரு Login/Register project செய்தால் Privacy Policy-ல்:
• We collect user name and email.
• We do not share user data with third parties.
• Passwords are encrypted before storing.
• User data is protected and stored securely.
Why Interviewers Like It?

பல freshers JWT, bcrypt, CORS மட்டும் சொல்வார்கள். ஆனால் நீ:

"To secure an application, I use JWT, password hashing, input validation, and also provide a Privacy Policy so users know how their data is handled."
02
Terms & Conditions
Definition
Terms and Conditions (T&C) என்றால் என்ன?

Terms and Conditions என்பது ஒரு website அல்லது application-ஐ பயன்படுத்தும் போது user பின்பற்ற வேண்டிய rules மற்றும் conditions-ஐ கூறும் document.

Easy Example Instagram account create பண்ணும்போது:
✔️ Fake account create பண்ணக்கூடாது
✔️ Illegal content post பண்ணக்கூடாது
✔️ Other users-ஐ abuse பண்ணக்கூடாது
இவை எல்லாம் Terms and Conditions-ல் இருக்கும்.
Privacy Policy vs Terms & Conditions
Privacy Policy Terms & Conditions
User data பற்றி பேசும் App usage rules பற்றி பேசும்
Data எப்படி collect & protect பண்ணுறோம் User என்ன செய்யலாம், என்ன செய்யக்கூடாது
Privacy focus Rules focus
Interview Answer

"Terms and Conditions define the rules and responsibilities for users when using an application. They explain what users are allowed to do, what is prohibited, and the responsibilities of both the user and the company."

Simple English

"Terms and Conditions are the rules that users must follow while using an application or website."

MERN Project Example

Suppose you build a Job Portal:

  • Users must provide correct information.
  • Users should not create fake accounts.
  • Users should not post illegal content.
  • The company can suspend accounts that violate the rules.
Interview Impression

Q: "How do you make your application professional and secure?"

"I use JWT authentication, password hashing, input validation, Privacy Policy for user data transparency, and Terms & Conditions to define user responsibilities and platform rules."
03
Audit Logs
Definition
Audit Logs என்றால் என்ன?

Audit Log என்பது application-ல் யார் என்ன action செய்தார்கள் என்பதை record செய்து வைத்திருக்கும் history.

எளிமையாக சொன்னால்:
"User or Admin என்ன செய்தார், எப்போது செய்தார் என்பதை track செய்து வைத்திருக்கும் log."
Example (Job Portal)
User Action Time
Rocky Login 10:00 AM
Rocky Applied for Job 10:05 AM
Admin Deleted Job Post 10:10 AM

இந்த records எல்லாம் Audit Logs.

Why is it Important?

Suppose: A user says "I didn't delete my account." An admin says "I didn't remove that job post."

Audit Logs பார்த்து:

  • Who did it?
  • When did it happen?
  • From which account?

என்பதை கண்டுபிடிக்கலாம்.

Interview Answer

"Audit Logs are records of important actions performed by users or administrators in an application. They help track activities, improve security, and make it easier to investigate issues."

MERN Example (JSON Structure)
User Login 
{
  "userId": "123",
  "action": "LOGIN",
  "time": "2026-06-02 10:00 AM"
}
Admin Delete User 
{
  "adminId": "456",
  "action": "DELETE_USER",
  "targetUser": "123",
  "time": "2026-06-02 10:30 AM"
}
Real Companies Use
  • Banking apps
  • E-commerce websites
  • Hospital systems
  • Admin dashboards
  • Government portals
Interview Impress

"Besides authentication and authorization, I also consider Audit Logs important because they help monitor user activities and improve security and accountability."

04
Security Headers
Definition
Security Headers என்றால் என்ன?

Security Headers என்பது browser-க்கு server அனுப்பும் special HTTP headers. இவை application-ஐ common attacks-லிருந்து பாதுகாக்க உதவும்.

எளிமையாக:
"Browser-க்கு security rules சொல்லும் HTTP headers."
Example Headers

User website open பண்ணும்போது:

Content-Security-Policy: default-src 'self'
X-Frame-Options: DENY
Strict-Transport-Security: max-age=31536000

இந்த headers browser-க்கு: எந்த content load பண்ணலாம், website iframe-ல் open ஆகக்கூடாதா, HTTPS மட்டும் use பண்ணணுமா என்று rules கொடுக்கும்.

1. Content-Security-Policy (CSP)

XSS attacks-ஐ குறைக்க உதவும்.

Content-Security-Policy: default-src 'self'

Meaning: நம்ம website content மட்டும் load ஆகும்.

2. X-Frame-Options

Clickjacking attack-ஐ தடுக்கிறது.

X-Frame-Options: DENY

Meaning: வேறு website iframe-ல் நம்ம site-ஐ embed பண்ண முடியாது.

3. Strict-Transport-Security

HTTPS மட்டும் use செய்ய browser-ஐ force பண்ணும்.

Strict-Transport-Security: max-age=31536000
4. X-Content-Type-Options

Browser file type guessing பண்ணாமல் தடுக்கிறது.

X-Content-Type-Options: nosniff
MERN Project-ல் எப்படி?

Express.js-ல் பொதுவாக Helmet.js package use பண்ணுவார்கள்.

npm install helmet
import helmet from "helmet";

app.use(helmet());

Helmet automatically many security headers add பண்ணும்.

Interview Answer

"Security Headers are HTTP response headers that help protect web applications from attacks such as XSS, clickjacking, and insecure connections. In Express applications, I can use Helmet.js to configure these headers easily."

Simple English

"Security Headers add extra protection to a web application by telling the browser how to handle security-related rules."

05
Rate Limiting
Definition
Rate Limiting என்றால் என்ன?

Rate Limiting என்பது ஒரு user அல்லது IP address ஒரு குறிப்பிட்ட நேரத்தில் எத்தனை requests அனுப்பலாம் என்று limit வைப்பது.

எளிமையாக:
"ஒரே user அதிக requests அனுப்புவதை கட்டுப்படுத்துவது."
Example: Login API
❌ Without Rate Limiting

Hacker 1 second-ல் 1000 passwords try பண்ணலாம்.

✅ With Rate Limiting

1 minute-க்கு 5 login attempts மட்டும்.
5 attempts முடிந்தால் temporary block.

Real-Life Example

ATM PIN: 3 தடவை wrong PIN போட்டால் card block ஆகும்.

அதே concept தான் Rate Limiting.

Why Use Rate Limiting?
  • ✅ Prevent Brute Force Attacks
  • ✅ Prevent Spam Requests
  • ✅ Protect Server Resources
  • ✅ Avoid API Abuse
  • ✅ Improve Application Security
MERN Example (Express.js)
npm install express-rate-limit
import rateLimit from "express-rate-limit";

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100
});

app.use(limiter);

Meaning: 15 minutes-க்கு 100 requests மட்டும். அதற்கு மேல் வந்தால் block.

Interview Answer

"Rate Limiting is a security technique that restricts the number of requests a user can make within a specific time period. It helps prevent brute-force attacks, spam, and server overload."

Simple English

"Rate Limiting controls how many requests a user can send to the server in a given time."

Interview Impress Answer

Q: "How do you secure a MERN application?"

"I use JWT authentication, bcrypt for password hashing, input validation, role-based access control, security headers with Helmet.js, rate limiting to prevent abuse, audit logs for tracking activities, and Privacy Policy and Terms & Conditions for compliance and user transparency."
