Key points மட்டும் நினைவில் வை:

  • JSX = JavaScript XML

  • Used to create UI in React

Virtual DOM

Virtual DOM is a lightweight copy of the Real DOM. When data changes, React first updates the Virtual DOM and compares it with the previous Virtual DOM. Then React updates only the changed parts in the Real DOM, which improves performance and makes React faster.

Important Keywords

  • Virtual DOM = Copy of Real DOM
  • Compare old and new Virtual DOM (Diffing)
  • Update only changed parts (Reconciliation)
  • Better Performance
  • React is Fast

component

A Component is a reusable part of the user interface in React. Instead of writing the same code again and again, we create a Component once and use it multiple times.

Props stands for Properties. Props are used to pass data from a parent component to a child component. They make components reusable by allowing different data to be passed.

  • Props = Properties
  • Parent ➝ Child Data Passing
  • Read-only
  • Reusable Components

State is used to store data inside a component. When the state changes, React automatically updates the UI without refreshing the page.

Keywords மட்டும் நினைவில் வை

  • Store Data
  • Inside Component
  • Dynamic Data
  • UI Updates Automatically
  • No Page Refresh
  • Re-render

Hooks

A Hook is a special function in React that allows functional components to use features like state and lifecycle methods. Hooks help manage state and handle side effects.

Keywords

  • Special Function
  • Functional Components
  • State (useState)
  • Lifecycle / Side Effects (useEffect)
  • No Need for Class Components

useState is a React Hook used to manage state in functional components. It allows us to store data and update the UI when the data changes.

Keywords

  • React Hook
  • Manage State
  • Store Data
  • Update UI
  • Functional Components

useEffect is a React Hook used to handle side effects in functional components. It is commonly used for API calls, data fetching, timers, and updating the DOM after rendering.

Keywords

  • React Hook
  • Side Effects
  • API Calls
  • Data Fetching
  • Timers
  • DOM Updates
  • Runs After Render

useContext is a React Hook used to share data between components without passing props manually through every level. It helps access shared data easily in multiple components

Keywords

  • React Hook
  • Share Data
  • Avoid Prop Drilling
  • Global/Shared Data
  • Easy Access

useRef is a React Hook used to access DOM elements directly. It can also store values that persist between renders. Updating a useRef value does not cause a component to re-render.

Keywords

  • React Hook
  • Access DOM elements directly
  • Store mutable values
  • No re-render when value changes

useMemo is a React Hook used to memoize (cache) a calculated value. It prevents expensive calculations from running on every render and helps improve performance.

அல்லது இன்னும் simple-ஆ:

useMemo stores a calculated value and reuses it until its dependencies change. This helps avoid unnecessary calculations and improves performance.

Key points:

  • useMemo is a React Hook.
  • It caches/stores a calculated value.
  • Prevents unnecessary recalculations.
  • Improves performance.

useCallback is a React Hook used to memoize a function. It prevents the function from being recreated on every render and helps improve performance.

Key points மட்டும்:

  • useCallback is a React Hook.
  • It memoizes (stores) a function.
  • Prevents function recreation on every render.
  • Helps avoid unnecessary child component re-renders.
  • Improves performance.

Component Lifecycle

Component Lifecycle refers to the different stages of a React component, from creation to removal. The main phases are Mounting, Updating, and Unmounting.

Key points:

  • Different stages of a component.
  • Mounting – Component is created.
  • Updating – Component is updated.
  • Unmounting – Component is removed.
  • Used to manage component behavior.

Mounting is the first phase of the React component lifecycle. It happens when a component is created and added to the DOM for the first time.

Key points:

  • First phase of Component Lifecycle.
  • Component is created.
  • Added to the DOM for the first time.
  • Initial setup tasks are performed.
  • Example: API calls, state initialization.

Updating is a phase in the React lifecycle where a component re-renders when its state or props change. It ensures that the UI displays the latest data.

அல்லது short-ஆ:

Updating happens when state or props change. React re-renders the component to update the UI with the latest data.

Key points:

  • Updating is a React lifecycle phase.
  • Happens when state or props change.
  • Component re-renders.
  • UI shows the latest data.

Unmounting is the final phase of the React component lifecycle. It happens when a component is removed from the DOM. During this phase, we clean up resources like timers, event listeners, and subscriptions.

Key points:

  • Final phase of React lifecycle.
  • Component is removed from the DOM.
  • Used for cleanup.
  • Removes timers, event listeners, subscriptions.
  • Prevents memory leaks.
