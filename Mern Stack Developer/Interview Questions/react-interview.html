01

Core Concepts

React
What is React?
React is a JavaScript library used for building user interfaces (UI). It helps developers create reusable components and build Single Page Applications (SPA).
Why React?
React is used because it provides reusable components, better code maintainability, fast rendering through Virtual DOM, and an efficient way to build modern web applications.
JSX
What is JSX?
JSX (JavaScript XML) is a syntax extension for JavaScript that allows us to write HTML-like code inside JavaScript. It is used in React to describe the UI.
Why do we use JSX?
We use JSX because it makes React code easier to write, read, and understand. It allows developers to create UI elements in a simpler and more readable way.
Virtual DOM
What is Virtual DOM?
Virtual DOM is a lightweight copy of the Real DOM maintained by React.
Why do we use Virtual DOM?
We use Virtual DOM to improve performance. Instead of updating the entire Real DOM, React first updates the Virtual DOM, compares the changes, and updates only the necessary parts in the Real DOM.
Component
What is a Component?
A Component is a reusable and independent piece of UI in React. It helps divide the application into smaller parts.
Why do we use Components?
We use Components to make code reusable, maintainable, and easier to manage. Instead of writing the same UI multiple times, we can create one component and reuse it wherever needed.
Props
What is Props?
Props (Properties) are used to pass data from a parent component to a child component in React.
Why do we use Props?
We use Props to share data between components and make components reusable with different values.
State
What is State?
State is used to store and manage data inside a React component.
Why do we use State?
We use State because when the data changes, React automatically updates the UI without refreshing the page.
02

UI Features

Event Handling
What is Event Handling?
Event Handling is used to handle user interactions like clicks, inputs, and form submissions in React.
Why do we use Event Handling?
We use it to perform actions based on user events and make the application interactive.
Conditional Rendering
What is Conditional Rendering?
Conditional Rendering is used to display different UI elements based on a condition in React.
Why do we use Conditional Rendering?
We use Conditional Rendering to show or hide content based on specific conditions or user actions.
Lists and Keys
What are Lists and Keys?
Lists are used to display multiple items dynamically in React, and Keys are unique identifiers used to identify each item in the list.
Why do we use Lists and Keys?
We use Lists to render multiple items efficiently and Keys to help React identify and update list items correctly.
List
What is List?
A List is used to display multiple items dynamically in React.
Why do we use List?
We use Lists to render and display a collection of data efficiently in the UI.
Form
What is Form?
A Form is used to collect user input such as name, email, password, and other details in a React application.
Why do we use Form?
We use Forms to collect, validate, and submit user data to the application or server.
Fragment
What is Fragment?
A Fragment is a React component that allows us to group multiple elements without adding an extra DOM element.
Why do we use Fragment?
We use Fragments to avoid unnecessary wrapper elements and keep the DOM structure clean.
03

Hooks

Hook
What is Hook?
A Hook is a special function in React that allows functional components to use React features such as state and lifecycle methods.
Why do we use Hooks?
We use Hooks to manage state, handle side effects, and add React functionality to functional components without using class components.
useState
What is useState?
useState is a React Hook used to store and manage state in a functional component.
Why do we use useState?
We use useState to handle dynamic data and update the UI whenever the state changes.
useEffect
What is useEffect?
useEffect is a React Hook used to perform side effects in a functional component.
Why do we use useEffect?
We use useEffect to handle operations such as API calls, data fetching, timers, and updating the DOM after the component renders.
useContext
What is useContext?
useContext is a React Hook used to access and share data across multiple components without passing props manually at every level.
Why do we use useContext?
We use useContext to avoid prop drilling and make shared data available to multiple components easily.
useRef
What is useRef?
useRef is a React Hook used to create a reference to a DOM element or store a mutable value without causing a re-render.
Why do we use useRef?
We use useRef to access DOM elements directly and store values that do not need to trigger a UI update.
useMemo
What is useMemo?
useMemo is a React Hook used to memoize and cache the result of an expensive calculation.
Why do we use useMemo?
We use useMemo to improve performance by preventing unnecessary recalculations during re-renders.
useCallback
What is useCallback?
useCallback is a React Hook used to memoize a function and prevent it from being recreated on every render.
Why do we use useCallback?
We use useCallback to improve performance by avoiding unnecessary function recreations and re-renders of child components.
Custom Hook
What is a Custom Hook?
A Custom Hook is a user-defined React Hook that allows us to reuse stateful logic across multiple components.
Why do we use Custom Hooks?
We use Custom Hooks to reuse common logic, reduce code duplication, and make the code more maintainable.
04

Component Lifecycle

Component Lifecycle
What is Component Lifecycle?
Component Lifecycle refers to the different stages a React component goes through from creation to updating and finally removal.
Why do we use Component Lifecycle?
We use Component Lifecycle to perform actions at specific stages of a component, such as fetching data, updating the UI, and cleaning up resources.
Mounting
What is Mounting?
Mounting is the process of creating and inserting a React component into the DOM for the first time.
Why do we use Mounting?
We use the mounting phase to perform initial tasks such as fetching data, setting up state, or initializing the component.
Updating
What is Updating?
Updating is the process that occurs when a component's state or props change, causing the component to re-render.
Why do we use Updating?
We use updating to reflect the latest data and changes in the UI.
Unmounting
What is Unmounting?
Unmounting is the process of removing a React component from the DOM.
Why do we use Unmounting?
We use unmounting to clean up resources such as timers, event listeners, and subscriptions before the component is removed.
useEffect Lifecycle Equivalent
What is useEffect Lifecycle Equivalent?
useEffect is a React Hook that can be used to handle component lifecycle events such as Mounting, Updating, and Unmounting in functional components.
Why do we use useEffect for Lifecycle?
We use useEffect to perform actions during different lifecycle stages without using class components.
