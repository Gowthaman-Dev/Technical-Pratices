Environment Variables

// Step-by-step classroom guide · MERN Stack · Node.js

1️⃣

Start with a Problem (Hook)

Hook
Teacher first question கேளுங்க:
"Database password அல்லது API key-ஐ code-ல direct எழுதலாமா?"
Example
bad-practice.js
const dbPassword = "mypassword123"
📌 Problem

• Code GitHub-க்கு push பண்ணினா
• Password public ஆகிடும் ❌

So security problem.

2️⃣

Introduce Environment Variables

Concept
📌 Tamil Explanation
Environment Variables என்பது
👉 Application configuration values store பண்ணும் variables
இந்த values code-க்கு வெளியே store ஆகும்
Examples
Database URL JWT Secret Port number API keys
3️⃣

Real Project Example

MERN
Teacher சொல்லணும்:
ஒரு MERN project-ல usually இந்த values இருக்கும்:
project values
Database URL
JWT secret
Server port
API keys

இந்த values code-ல எழுதக்கூடாது.

அதனால .env file use பண்ணுறோம்

4️⃣

.env File Explanation

File

Project root-ல .env file create பண்ணுவோம்.

Example:

.env
PORT=5000
MONGO_URL=mongodb://localhost:27017/jobportal
JWT_SECRET=mysecretkey
📌 Explanation
PORT
→ server port
MONGO_URL
→ database connection
JWT_SECRET
→ authentication secret
5️⃣

dotenv Package

Package
Node.js .env file read பண்ண முடியாது
அதனால dotenv package use பண்ணுறோம்.
Install
terminal
npm install dotenv
6️⃣

Server File Setup

Code
server.js
import dotenv from "dotenv"

dotenv.config()

const port = process.env.PORT

app.listen(port, () => {
 console.log("Server running")
})
📌 Explanation
dotenv.config()
→ .env variables load ஆகும்
process.env
→ environment variables access
7️⃣

Access Environment Variables

Access

Node.js-ல் variables access பண்ண:

access
process.env.PORT
process.env.JWT_SECRET
process.env.MONGO_URL
Example
usage
const secret = process.env.JWT_SECRET

9️⃣

Why Environment Variables Used

Reasons
1️⃣ Security

Secret values code-ல இருக்காது.

Example:

JWT_SECRET API_KEY DB_PASSWORD
2️⃣ Easy Configuration

Port change பண்ண easy.

PORT=5000
PORT=8000

Code change தேவையில்லை.

3️⃣ Different Environments

Projects-ல usually 3 environments இருக்கும்:

Development Testing Production

ஒவ்வொன்றுக்கும் different configuration

Example:

DEV_DB_URL
PROD_DB_URL

Flow

Interview Answers

Interview

Environment variables are used to store configuration values like database URLs, API keys, and secrets outside the source code.
⭐ Better Interview Answer (Professional)
Environment variables allow us to store sensitive configuration values such as database URLs, API keys, and JWT secrets outside the application code, improving security and flexibility.
