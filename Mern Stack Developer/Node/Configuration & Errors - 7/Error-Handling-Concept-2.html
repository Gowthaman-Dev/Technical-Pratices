2️⃣

Method 1 – Try Catch

Method 1
try...catch
📌 Purpose
Async operations errors handle பண்ண.
Example
try-catch.js
try {
 const user = await User.find()
}
catch(error){
 console.log(error)
}
Explanation (Class-ல சொல்ல)
try
→ risky code
catch
→ error handle
Example flow
Database Query
↓
Error Occurs
↓
Catch Block Handles
3️⃣

Method 2 – Send Error Response

Method 2
HTTP Response

Server error வந்தா client-க்கு proper response அனுப்பணும்.

Example
error-response.js
catch(error){
 res.status(500).json({message:"Server Error"})
}
Explanation
status(500)
→ server error
json()
→ response send
Client side output
Output
{
 "message": "Server Error"
}
4️⃣

Method 3 – Global Error Middleware

Method 3
Global Middleware

Express-ல் central error handler create பண்ணலாம்.

Example
errorMiddleware.js
app.use((err,req,res,next)=>{
 res.status(500).json({
  message: err.message
 })
})
Explanation
err
→ error object
req
→ request
res
→ response
next
→ next middleware
Flow
Controller Error
↓
Global Middleware
↓
Response Send
📌 Advantage
👉 All errors one place-ல handle ஆகும்
5️⃣

Controller Example (Class)

Code
userController.js
export const getUsers = async (req,res)=>{
 try{
   const users = await User.find()
   res.json(users)
 }
 catch(err){
   res.status(500).json({message:"Error fetching users"})
 }
}
Explanation
try
→ database query
catch
→ error response
