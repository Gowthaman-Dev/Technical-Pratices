Environment Variables

🔐
📌 Concept (Tamil)

Environment Variables என்பது 👉 application configuration values store பண்ணும் variables.
இதுல usually secret values இருக்கும்.

Examples
  • Database URL
  • JWT Secret
  • Port number
  • API keys

இந்த values codeல direct எழுதக்கூடாது ❌ அதற்கு பதில் .env file use பண்ணுவோம்.

📌 Example `.env`
.env 
PORT=5000
MONGO_URL=mongodb://localhost:27017/jobportal
JWT_SECRET=mysecretkey
📌 dotenv package
Info

Node.js-ல் .env variables load பண்ண dotenv use பண்ணுவோம்.

terminal — Install 
npm install dotenv
📌 Server File
server.js 
import dotenv from "dotenv"

dotenv.config()

const port = process.env.PORT

app.listen(port, ()=>{
 console.log("Server running")
})
📌 Access Environment Variable
access via process.env 
process.env.PORT
process.env.JWT_SECRET
process.env.MONGO_URL
📌 Why Environment Variables Used
  • 🔒
    1️⃣ Securitysecret values safe
  • ⚙️
    2️⃣ Configuration change easyno code change needed
  • 🌐
    3️⃣ Different environmentsDevelopment · Production · Testing
📌 Interview English Sentence

Environment variables are used to store configuration values like database URLs, API keys, and secrets outside the source code.

Error Handling

🛡️
📌 Concept (Tamil)

Application run ஆகும்போது errors occur ஆகும்.
இந்த errors handle பண்ணுவது Error Handling.

Examples
  • Database connection error
  • Invalid request
  • Server crash
  • API failure
Types of Errors
1️⃣ Syntax Error
Code mistake. Node run ஆகாது. const name =
2️⃣ Runtime Error
Program run ஆகும்போது error வரும். user undefined. user.name
3️⃣ Logical Error
Code run ஆகும் ஆனால் output wrong. Wrong calculation. 2 + 2 = 5
Express Error Handling Methods
1️⃣ Try Catch

Async code error handle பண்ண.

try-catch 
try {
 const user = await User.find()
}
catch(error){
 console.log(error)
}
2️⃣ Send Error Response

Client-க்கு error response அனுப்பு.

error response 
catch(error){
 res.status(500).json({
  message:"Server Error"
 })
}
3️⃣ Global Error Middleware

Express-ல் central error handler create பண்ணலாம்.

middleware.js 
app.use((err,req,res,next)=>{
 res.status(500).json({
  message: err.message
 })
})
📌 Example Controller
userController.js 
export const getUsers = async (req,res)=>{
 try{
   const users = await User.find()
   res.json(users)
 }
 catch(err){
   res.status(500).json({message:"Error fetching users"})
 }
}

Interviewer கேட்கும் Common Questions

💬
  • Q1What are environment variables?
    Environment variables store configuration values like database URLs and secret keys outside the source code.
  • Q2Why use dotenv?
    We use dotenv to load environment variables from a .env file into process.env.
  • Q3How do you handle errors in Express?
    We use try-catch blocks in async functions and also implement global error handling middleware.
  • Q4How do you access environment variables?
    Environment variables are accessed using process.env.
