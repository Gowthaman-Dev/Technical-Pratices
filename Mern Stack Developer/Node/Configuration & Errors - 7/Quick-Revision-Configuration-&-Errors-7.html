Environment Variables

Definition
Environment Variables

Application configuration values store பண்ணும் variables. இந்த values code-க்கு வெளியே store ஆகும். இதுல usually secret values இருக்கும் — code-ல் direct எழுதக்கூடாது.

Database URL JWT Secret Port number API keys
Bad Practice — Never do this
const dbPassword = "mypassword123"  // ❌ WRONG

Code GitHub-க்கு push பண்ணினா → Password public ஆகிடும். இதனால் .env file use பண்ணுவோம்.

.env File Example
.env
PORT=5000
MONGO_URL=mongodb://localhost:27017/jobportal
JWT_SECRET=mysecretkey
VariablePurpose
PORTServer port number
MONGO_URLDatabase connection string
JWT_SECRETAuthentication secret key
dotenv Package — Setup
Install
npm install dotenv

Node.js .env file directly read பண்ண முடியாது — அதனால் dotenv package use பண்ணுறோம்.

server.js
import dotenv from "dotenv"

dotenv.config()  // .env load ஆகும்

const port = process.env.PORT

app.listen(port, () => {
  console.log("Server running")
})

Why Environment Variables Used
1
Security
Secret values code-ல் இருக்காது. GitHub push பண்ணினாலும் safe.
JWT_SECRET API_KEY DB_PASSWORD
2
Easy Configuration
Port change பண்ண easy. Code change தேவையில்லை — .env மட்டும் change பண்ணினால் போதும்.
PORT=5000 PORT=8000
3
Different Environments
ஒவ்வொரு environment-க்கும் different configuration.
Development Testing Production

Example: DEV_DB_URL vs PROD_DB_URL — same code, different values per environment.

Error Handling

Error Handling

Definition
Error Handling

Application run ஆகும்போது வரும் errors-ஐ detect பண்ணி handle பண்ணுற process. Error handle பண்ணவில்லை என்றால் server crash ஆகும்.

Examples: Database connection error · Invalid API request · Server crash · API failure · Authentication failure

3 Types of Errors
1
Syntax Error
Code syntax mistake. Program run ஆகவே முடியாது.
Example
const name =
SyntaxError: Unexpected end
2
Runtime Error
Program run ஆகும்போது error வரும். API calls, DB queries-ல் occur ஆகும்.
Example
const user = undefined
console.log(user.name)
TypeError: Cannot read 'name'
3
Logical Error
Code run ஆகும் ஆனால் wrong output வரும். Debugging மூலம் கண்டுபிடிக்கணும்.
Example
const result = 2 + 2
console.log(result === 5)
Expected 4, logic wrong → 5

3 Express Error Handling Methods
Method 1Try Catch

Async operations errors handle பண்ண.

try {
  const user = await User.find()
} catch(error) {
  console.log(error)  // try → risky code · catch → error handle
}
Method 2Send Error Response

Server error வந்தா client-க்கு proper HTTP response அனுப்பணும்.

catch(error) {
  res.status(500).json({ message: "Server Error" })
  // status(500) → server error · json() → response send
}
Method 3Global Error Middleware

Express-ல் central error handler. All errors one place-ல் handle ஆகும்.

// errorMiddleware.js
app.use((err, req, res, next) => {
  res.status(500).json({ message: err.message })
  // err→error · req→request · res→response · next→next middleware
})
Controller Example — Real MERN
userController.js
export const getUsers = async (req, res) => {
  try {
    const users = await User.find()
    res.json(users)
  } catch(err) {
    res.status(500).json({
      message: "Error fetching users"
    })
  }
}
authController.js (interview level)
export const register = async (req, res) => {
  try {
    const user = await User.create(req.body)
    res.status(201).json(user)
  } catch(error) {
    res.status(500).json({
      message: "Registration failed"
    })
  }
}

Common Interview Questions
Q1: What are environment variables?
Environment variables store configuration values like database URLs and secret keys outside the source code.
Q2: Why use dotenv?
We use dotenv to load environment variables from a .env file into process.env.
Q3: How do you handle errors in Express?
We use try-catch blocks in async functions and also implement global error handling middleware.
Q4: How do you access environment variables?
Environment variables are accessed using process.env.
