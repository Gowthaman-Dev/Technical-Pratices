Event Loop – Class Guide

Step-by-step teaching · Tamil + English

1️⃣ Step 1 Start With Question
Class start பண்ணும்போது கேள்:
🗣️ "Node.js single thread தான்… அப்போ எப்படி thousands of users handle பண்ணும்?"
Students usually answer தெரியாது.
👉 இதற்கு reason Event Loop.
2️⃣ Step 2 Simple Definition
Board-ல எழுதணும்:
Definition
👉 Event Loop = Node.js asynchronous tasks manage பண்ணும் system
Simple meaning:
👉 Node.js single thread இருந்தாலும் multiple tasks handle பண்ண உதவும் mechanism.
3️⃣ Step 3 Event Loop Components
Board-ல diagram draw பண்ணு.
Call Stack
↓
Node APIs
↓
Callback Queue
↓
Event Loop
4️⃣ Step 4 Each Component Explain
1️⃣ Call Stack
👉 Current code execute ஆகும் place.
Example:
console.log("Hello")

இந்த code Call Stack-ல run ஆகும்.

2️⃣ Node APIs

Async tasks run ஆகும் place.

Examples:
setTimeout fs.readFile API request
Example:
setTimeout(()=>{},2000)
👉 இது Node API-க்கு போகும்.
3️⃣ Callback Queue

Async task complete ஆன பிறகு callback wait பண்ணும் place.

Example callback:
()=>{
 console.log("Async task")
}
4️⃣ Event Loop

Event loop continuously check பண்ணும்:

Call stack empty ?
If yes:
👉 Callback queue → call stack move பண்ணும்
அப்புறம் execute ஆகும்.
5️⃣ Step 5 Code Example
Students-க்கு இந்த code காட்டணும்.
console.log("Start")

setTimeout(()=>{
  console.log("Async task")
},2000)

console.log("End")

9️⃣ Why Event Loop Important
Board-ல explain:
✅ Scalable Node.js scalable
✅ Non-Blocking non-blocking operations
✅ High Volume thousands of users handle
இதனால தான் Node.js high performance server.
