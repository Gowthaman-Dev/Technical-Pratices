01 Callback Functions
Definition
Callback Function
ஒரு function-க்கு argument ஆக pass பண்ணப்படும் function — அந்த operation முடிந்த பிறகு execute ஆகும். Node.js-ல async operations handle பண்ண அதிகமாக use பண்ணப்பட்டது.
File read API call Database query setTimeout
Simple Example
function greet(name, callback) { console.log("Hello " + name) callback() // pass ஆன function call பண்ணும் } function done() { console.log("Task completed") } greet("Gowtham", done)
greet() run ஆகும்
"Hello Gowtham" print
callback() → done() execute
"Task completed" print
Output: 1Hello Gowtham 2Task completed
Real Node.js Example — fs.readFile
const fs = require("fs") fs.readFile("data.txt", "utf8", (err, data) => { if (err) { console.log(err) // error handle } else { console.log(data) // success } })
fs.readFile() → async — Node.js wait பண்ணாது
File read complete ஆகும்
Callback (err, data) => {} run ஆகும்

இதனால் தான் callback use பண்ணுறோம் — async task முடிந்த பிறகு மட்டுமே callback run ஆகும்.

Callback Hell
login(user, function() { getData(function() { updateData(function() { saveData(function() { // deeply nested... Pyramid of Doom }) }) }) })

Function inside function inside function → இதுதான் Callback Hell. Pyramid of Doom என்றும் சொல்வார்கள்.

Readability
Code read பண்ண கஷ்டம்
Debugging
Error trace difficult
Maintain
Change பண்ண hard
Solution to Callback Hell
Promises
login() .then(getData) .then(updateData) .then(saveData) .catch(err => ...)
Async / Await
async function run() { await login() await getData() await saveData() }

Modern Node.js mostly Promises / Async Await use பண்ணும் — Callbacks avoid பண்ணலாம்.

Callback Hell — Interview One Line
"Callback hell occurs when multiple callbacks are nested inside each other, making the code difficult to read and maintain."
Interview Sentence — Memorize
"A callback function is a function passed as an argument to another function and executed after the operation completes."
02 Promises
Definition
Promise
Asynchronous operation-ஐ handle பண்ண modern method. Future-ல் success அல்லது failure result return பண்ணும் object. Callback hell-ஐ solve பண்ண வந்தது.
3 States
Pending
Operation running
Fulfilled
Success
Rejected
Error / failure

Order placed → Pending · Food delivered → Fulfilled · Restaurant closed → Rejected

Promise Syntax
const promise = new Promise((resolve, reject) => { if (true) { resolve("Success") // Fulfilled } else { reject("Error") // Rejected } })
WordMeaning
new PromiseAsync task create பண்ணும்
resolveSuccess result pass பண்ணும்
rejectError pass பண்ணும்
Using Promise — .then() & .catch()
promise .then(result => { console.log(result) // Success handle }) .catch(error => { console.log(error) // Error handle })
MethodPurpose
.then()Promise fulfilled ஆனால் run ஆகும்
.catch()Promise rejected ஆனால் run ஆகும்
Real Async Example — setTimeout with Promise
const fetchData = new Promise((resolve, reject) => { setTimeout(() => { resolve("Data received") }, 2000) }) fetchData .then(data => console.log(data)) .catch(err => console.log(err))
Promise start → Pending
2 seconds wait → setTimeout Node API
resolve() → Fulfilled → .then() run
Output: Data received

Follow-up Interview Q — Promise vs Async/Await?
Promise → .then() chaining, functional style, moderate readability.
Async/Await → Promise-மேலே built, synchronous-like syntax, very readable, try/catch error handling.
Both same result — Async/Await is cleaner syntax for the same Promise underneath.
Interview Sentence — Memorize
"A Promise is an object that represents the eventual completion or failure of an asynchronous operation, with three states: pending, fulfilled, and rejected."
03 Async / Await
Definition
Async / Await
Promises-ஐ easy syntax-ல் handle பண்ண modern method. Code-ஐ synchronous style போல் readable-ஆ எழுத உதவும். Promises-ஐ replace பண்ணவில்லை — மேலே built ஆனது.
KeywordMeaning
asyncFunction always Promise return பண்ணும்
awaitPromise resolve ஆகும் வரை wait பண்ணும் — async function-ல் மட்டுமே use பண்ணலாம்
async Function
async function getData() { return "Hello" // automatically Promise wrap ஆகும் } getData().then(data => console.log(data))
Output: Hello
await Keyword
async function fetchData() { const data = await Promise.resolve("Data received") console.log(data) } fetchData()
fetchData start
await → promise resolve ஆகும் வரை wait
data received → console.log
Real Example — API Call with Error Handling
async function getUsers() { try { const res = await fetch("https://api.example.com/users") const data = await res.json() console.log(data) } catch(err) { console.log(err) // promise reject → catch run } }
CodeMeaning

Follow-up Interview Questions
Q1: await-ஐ async இல்லாமல் use பண்ணலாமா?
No — await-ஐ async function-ல் மட்டுமே use பண்ண முடியும். Outside use பண்ணினால் SyntaxError வரும்.

Q2: await மறந்தால் என்ன ஆகும்?
Promise object return ஆகும் — resolved value கிடைக்காது. Code wrong result தரும், error வராது.
Interview Sentence — Memorize
"Async/Await is a modern syntax built on top of Promises that allows writing asynchronous code in a synchronous style, making it more readable and easier to debug."
04 Event Loop
Flow
Call Stack → Node APIs → Callback Queue → Event Loop
4 Components
Call Stack
Current code execute ஆகும் இடம். Sync code இங்கே run ஆகும்.
Node APIs
Async tasks run ஆகும் இடம். setTimeout, fs, network calls.
Callback Queue
Async task முடிந்த பிறகு callback wait பண்ணும் இடம்.
Event Loop
Call stack empty? → Queue-ல் callback-ஐ எடுத்து execute பண்ணும்.
Code Example
console.log("Start") setTimeout(() => { console.log("Async task") }, 2000) console.log("End")
Output: 1Start 2End 3Async task (2s later)

setTimeout → Node API போகும் → Call stack clear ஆகும் → End print → 2s → callback queue → event loop → execute

Why Event Loop?
Scalable
Node.js scalable ஆக இருக்கும்
Non-Blocking
Operations block ஆகாது
High Volume
1000s of users handle
Real Life — Restaurant
Waiter = Event Loop · Order = Request · Chef = Node API
Waiter order wait பண்ண மாட்டான் → next customer போவான்.
Chef ready ஆனதும் waiter serve பண்ணுவான். இதுதான் Event Loop.
Interview Q — Call Stack vs Callback Queue
Call Stack
Sync code execute ஆகும் இடம்
LIFO order
Empty ஆகும் வரை run ஆகும்
Callback Queue
Async callbacks wait பண்ணும் இடம்
FIFO order
Call stack empty ஆனதும் execute
Interview Sentence — Memorize
"The event loop is the mechanism that allows Node.js to perform non-blocking asynchronous operations by handling callbacks from the event queue."
05 setTimeout / setInterval

setTimeout

06 setImmediate()
Definition
setImmediate()
Current I/O operations முடிந்த உடனே — event loop-ன் next cycle-ல் callback execute பண்ணும். No delay. No repeat.
Syntax & Basic Example
setImmediate(() => { console.log("Immediate task") })

No delay argument தேவை இல்லை — event loop next cycle-ல் automatically run ஆகும்.

Execution Flow
console.log("Start") setImmediate(() => { console.log("Immediate") }) console.log("End")
Output: 1Start 2End 3Immediate

Sync code முடிந்த பிறகு → event loop → setImmediate callback execute ஆகும்.

I/O என்ன? (Input / Output)

I/O

setImmediate → I/O phase முடிந்த உடனே run ஆகும் → இதனால் I/O callbacks-க்கு பிறகு queue-ல் execute ஆகும்.

Event Loop Connection
setImmediate() → Callback Queue → Event Loop check → Execute

Call Stack empty ஆனதும் → event loop → setImmediate callback run ஆகும்.

setTimeout(0) vs setImmediate — Key Interview Point
setTimeout(() => { console.log("Timeout") }, 0) setImmediate(() => { console.log("Immediate") })
Usually: 1Immediate 2Timeout

setImmediate → I/O phase-க்கு பிறகு run ஆகும். setTimeout(0) → timer phase-ல் run ஆகும். Order environment-ஐ பொறுத்து change ஆகலாம் — but I/O-க்கு உள்ளே இருந்தால் setImmediate always first.

Interview Q — process.nextTick() vs setImmediate()?
process.nextTick() → current operation முடிந்த உடனே, I/O-க்கும் முன்னே run ஆகும் (highest priority).
setImmediate() → I/O phase முடிந்த பிறகு, next event loop iteration-ல் run ஆகும்.
Interview Sentence — Memorize
"setImmediate executes a callback immediately after the current I/O operations are completed, in the next iteration of the event loop."
07 process.nextTick()
Definition
process.nextTick()
Current operation முடிந்த உடனே — event loop continue ஆவதற்கு முன்னாடியே — callback execute பண்ணும். Queue-க்கே போகாது. Fastest execution.
Syntax & Example
process.nextTick(() => { console.log("Next tick") })
Execution Flow
console.log("Start") process.nextTick(() => { console.log("Next tick") }) console.log("End")
Output: 1Start 2End 3Next tick

Sync code முடிந்த உடனே → event loop-க்கு முன்னே → nextTick callback run ஆகும்.

Where it runs in the Event Loop
Current code → nextTick queue → Event Loop → Callback Queue

nextTick queue → Event Loop-க்கே போகாமல் முன்னே run ஆகும் — இதனால் highest priority.

Execution Priority Order

Fastest — before event loop

After I/O phase
3 setTimeout()
Timer phase

Why use carefully? — Interview Q
process.nextTick-ஐ recursive-ஆக call பண்ணினால் → nextTick queue never empty ஆகாது → Event Loop starve ஆகும் → I/O, setTimeout எதுவும் run ஆகாது. இதனால் careful-ஆக use பண்ணணும்.
Interview Sentence — Memorize
"process.nextTick schedules a callback to be executed immediately after the current operation, before the event loop continues — making it the highest priority async callback in Node.js."

Ultimate Interview Summary

"Node.js handles asynchronous programming using callbacks, promises, and async/await. The event loop manages these tasks by processing callbacks from the queue without blocking the main thread. Additional mechanisms like setTimeout, setImmediate, and process.nextTick help schedule asynchronous execution."
