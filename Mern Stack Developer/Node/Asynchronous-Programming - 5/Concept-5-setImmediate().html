Part 1 · setImmediate()
1️⃣ Step 1 Start With Question
Class start பண்ணும்போது கேள்:
🗣️ "Code finish ஆன உடனே next task run பண்ணணும்னா எப்படி?"
Example:
current task finish
↓
next task run immediately
👉 Node.js-ல அதற்கு setImmediate() use பண்ணலாம்.
2️⃣ Step 2 Simple Definition
Board-ல எழுதணும்:
Definition
👉 setImmediate() = current I/O operations முடிந்த உடனே callback execute பண்ணும்
Simple meaning:
Current task finish
↓
Immediate callback run
3️⃣ Step 3 Basic Syntax 
setImmediate(function)
Example:
setImmediate(()=>{
 console.log("Immediate task")
})
Explain:
👉 event loop next cycle-ல run ஆகும்.

 console.log("Immediate")
})

console.log("End")

Start
End
Immediate
Explain:
1️⃣ Start run
2️⃣ setImmediate register ஆகும்
3️⃣ End run
4️⃣ Event loop → Immediate callback run

7️⃣ Step 7 Compare With setTimeout(0)
Important interview point.
Example:
setTimeout(()=>{
 console.log("Timeout")
},0)

setImmediate(()=>{
 console.log("Immediate")
})
Explain:
👉 execution order environment-ஐ பொறுத்து change ஆகலாம்.
But normally:
Immediate
Timeout
Reason:
👉 setImmediate I/O phase after execute ஆகும்.

Part 2 · I/O & Event Loop Deep Dive
1️⃣ First I/O என்ன?
I/O = Input / Output
Computer வெளியுலகத்துடன் communicate பண்ணும் operations.
Examples:
OperationType
File read I/O
Database queryI/O
API request I/O
Network requestI/O
Example code:
const fs = require("fs")

fs.readFile("data.txt","utf8",(err,data)=>{
 console.log(data)
})
Explain:
File read → I/O operation
ஏன்?
👉 Node.js disk-ல இருந்து data read பண்ணுது.

Call Stack

Node APIs

5️⃣ Callback Queue
Callback Queue

Async task complete ஆன பிறகு
callback queue-ல wait பண்ணும்.

Example callback:
()=>{
 console.log("Async task")
}
Flow:
Timer finished
↓
Callback queue
6️⃣ Event Loop
Event Loop

Event loop continuously check பண்ணும்:

Call stack empty ?
If YES:
Queue → Call stack
callback execute ஆகும்.
