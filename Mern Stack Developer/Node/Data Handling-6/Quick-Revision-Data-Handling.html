Definition & Why Streams
Stream — Data in small chunks

Data-ஐ small chunks (பகுதிகள்) ஆக process பண்ணும் method. Large files handle பண்ணும்போது முழு file memory-க்கு load பண்ணாமல் piece by piece process பண்ணும். அதனால் Node.js high performance.

4 Types of Streams
01 · READABLE
Readable Stream
Data read பண்ண use பண்ணும் stream.
File reading HTTP request Video streaming
02 · WRITABLE
Writable Stream
Data write பண்ண use பண்ணும் stream.
File writing HTTP response Logging
03 · DUPLEX
Duplex Stream
Read + Write இரண்டும் செய்யும் stream.
TCP socket Chat app WebSocket
04 · TRANSFORM
Transform Stream
Read → Modify → Write. Data transform செய்யும்.
Compression Encryption Data conversion

Remember: Readable → Read data · Writable → Write data · Duplex → Both · Transform → Read + Change + Write

Code Examples

pipe() — Readable → Writable connect
const readStream  = fs.createReadStream("data.txt")

readStream.pipe(writeStream)  // data.txt → copy.txt — one line!

Stream Events — Event-Based System
data
Chunk receive ஆனால் trigger ஆகும். Most important event.

end
File reading complete ஆன பிறகு run ஆகும்.

error

Writable stream writing complete ஆன பிறகு run ஆகும்.
writeStream.on("finish", () => {...})

data → receive · end → read finish · error → problem · finish → write finish

Backpressure — Flow Control (Interview Important)
What is Backpressure?

Data producer fast-ஆ send பண்ணும்போது consumer slow-ஆ process பண்ணும் situation. Water tank fast → Glass slow → Overflow ஆகும். இதுதான் Backpressure concept.

Problem

Read Stream (fast) → Write Stream (slow) → Data accumulate ஆகும்

Solution

1. Buffering — Extra data temp memory-ல் store
2. pipe() — Automatic pause/resume flow control

// pipe() automatically handles backpressure
readStream.pipe(writeStream)  // auto pause when slow
Flowing / Paused Mode
Flowing Mode
Automatic data flow
data event attach பண்ணினால் stream automatically flowing mode-க்கு போகும். Data automatic-ஆ வரும்.
readStream.on("data", (chunk) => {...})
Paused Mode
Manual control
Data automatic-ஆ வராது. Program தான் control பண்ணும். pause() / resume() use பண்ணலாம்.
readStream.pause() // stop
readStream.resume() // continue

Tap analogy: Flowing = tap open (auto flow) · Paused = tap close (manual control)

HighWaterMark — Buffer Size Control
HighWaterMark = chunk size limit

Stream buffer-ல் maximum data size எவ்வளவு இருக்கணும் என்று control பண்ணும் option. Default 64KB.

const stream = fs.createReadStream("data.txt", {
  highWaterMark: 16 * 1024  // 16KB chunk size
})
// Each chunk = 16KB — predictable size

Buffer overflow avoid · Memory control · Better performance · Flow control help

Final Interview Answer — Memorize

"Streams in Node.js are used to handle large data efficiently by processing it in small chunks instead of loading the entire data into memory. There are four types of streams: readable, writable, duplex, and transform. Streams are event-based and use events like data, end, and error. They also manage flow control using mechanisms like backpressure."

Buffers

Node.js Buffers

Definition
Buffer — Temporary Binary Memory

Binary data store பண்ணும் temporary memory area. Node.js-ல் files, images, videos போன்ற binary data handle பண்ண buffers use பண்ணப்படும். Computer எல்லா data-யும் binary (0 & 1) format-ல் தான் store பண்ணும்.

File reading Image processing Video streaming Network packets
3 Buffer Creation Methods
from existing
Buffer.from()
Existing data-இல் இருந்து buffer create பண்ணும்.
Example

// <Buffer 48 65 6c 6c 6f>
fixed size
Buffer.alloc()
Fixed size empty memory create பண்ணும்.
Example

// 10 bytes empty
fast (unsafe)
Buffer.allocUnsafe()
Fast memory allocate பண்ணும் — but unsafe (old data இருக்கும்).
Example
