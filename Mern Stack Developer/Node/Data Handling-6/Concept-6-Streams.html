Stream Events — Event-Based System

Node.js streams event-based system.

Stream-ல் ஏதாவது action நடந்தா event trigger ஆகும்.
Data வந்தது → data event  |  Reading முடிந்தது → end event
Error வந்தது → error event  |  Writing முடிந்தது → finish event
EventPurpose
datachunk receive
endreading finished
errorerror handling
finishwriting finished
data Event — Most Important

👉 File-ல இருந்து chunk வந்ததும் run ஆகும்.

const fs = require("fs") const readStream = fs.createReadStream("data.txt") readStream.on("data", (chunk) => { console.log(chunk) })
Chunk1 → data event  |  Chunk2 → data event  |  Chunk3 → data event
end Event

👉 File reading complete ஆன பிறகு run ஆகும்.

👉 File read செய்யும்போது error வந்தா run ஆகும்.

"Streams in Node.js are event-based. Important stream events include data, end, error, and finish. The data event is triggered when a chunk of data is received, end indicates reading completion, error handles failures, and finish indicates writing completion."

Backpressure — Flow Control

🚰 Water Pipe Example

Water Tank (fast)
↓
Pipe
↓
Glass (slow)
Problem: Tank fast-ஆ water send பண்ணுது → Glass slow-ஆ fill ஆகுது → Water overflow ஆகும்.
இதுதான் Backpressure concept.

2️⃣ Technical Explanation

Backpressure என்பது:

👉 Data producer
Fast Data Source
👉 Data consumer
Slow Destination
File
↓
Read Stream (fast)
↓
Write Stream (slow)
↓
Disk
👉 Write slow இருந்தா data accumulate ஆகும். இதுதான் Backpressure.

4️⃣ Streams Solution

Node.js streams automatically control flow. Two solutions:

1️⃣ Buffering
Extra data temporary memory-ல store ஆகும்
Fast Source
↓
Buffer
↓
Slow Destination
2️⃣ pipe() Flow Control
pipe() automatically manage பண்ணும்.
readStream.pipe(writeStream) // pipe() automatically pause/resume streams
Producer fast Consumer slow = Backpressure Solution: Buffer + pipe() + Flow control

"Backpressure occurs when the data producer is faster than the consumer. Node.js streams handle this situation using buffering and flow control mechanisms like pipe()."
