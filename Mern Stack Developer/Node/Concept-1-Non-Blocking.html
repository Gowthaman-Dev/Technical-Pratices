Step 2 – Blocking Concept Explain

Board-ல example draw பண்ணு.

Read File
↓
(wait)
↓
Next Code

Explain:

🗣️

"Traditional languages (Java, PHP synchronous code)-ல file read முடியும் வரை server wait பண்ணும்."

Example flow

1
Read file
2
Wait

Next operation

Problem

  • server slow
  • requests queue ஆகும்
  • scalability குறையும்
03

Step 3 – Node.js Idea Introduce

அடுத்த question:

🗣️

"Wait பண்ணாமல் next work செய்ய முடியுமா?"

Answer

👉 Node.js அதையே செய்கிறது.

Explain

👉 Node.js Non-Blocking I/O use பண்ணும்.

Meaning

Task start பண்ணி → wait பண்ணாமல் → next code execute பண்ணும்

04

Step 4 – Code Example (Most Important)

Students-க்கு code show பண்ணு.

non-blocking.js 
const fs = require("fs");

fs.readFile("data.txt", "utf8", (err, data) => {
  console.log(data);
});

console.log("File reading started");

Explain line by line:

Line 1

const fs = require("fs")

👉 File System module import.

Line 2

fs.readFile()

👉 file read operation start ஆகுது.

Callback function

(err, data) => {}

👉 file read complete ஆன பிறகு run ஆகும்.

Next line

console.log("File reading started")

👉 Node.js wait பண்ணாமல் next code execute பண்ணும்.

1️⃣

Deep Dive – I/O என்றால் என்ன?
1️⃣

I/O என்றால் என்ன?

I/O = Input / Output

Meaning

  • Computer வெளியில் இருந்து data வாங்குவது (Input)
  • Computer data வெளியே அனுப்புவது (Output)
2️⃣

Simple Example

Input

நீ keyboard-ல type பண்ணுறது

👉 Input

Output

Screen-ல text காட்டுவது

👉 Output

இதுதான் I/O.

3️⃣

Programming-ல I/O என்ன?

Program external source-லிருந்து data வாங்கினால் அல்லது data send பண்ணினால் அது I/O.

Examples

1️⃣

File Read

Program file open பண்ணி data read பண்ணும்.

Example

file-read.js 
fs.readFile("data.txt")
  • file-லிருந்து data வாங்குறது
  • இது Input
2️⃣

Database Query

Database-ல data கேட்குறது.

Example

database.js 
User.find()
  • database-லிருந்து data வரும்
  • இது I/O
3️⃣

API Call

Server இன்னொரு server-க்கு request அனுப்பும்.

Example

api-call.js 
axios.get("/users")
  • external API-லிருந்து data வரும்
  • இது I/O
4️⃣

Network Request

Browser server-க்கு request அனுப்பும்.

Example

network.js 
fetch("/products")
  • network-ல data travel ஆகுது
  • இது I/O
4️⃣

Simple Definition (Remember)

👉 Computer external source-லிருந்து data read அல்லது write பண்ணும் operations = I/O

External sources

File system Database Network API
5️⃣

Why I/O Important in Node.js

Node.js I/O operations wait பண்ணாது.

Example

non-blocking.js 
fs.readFile()

File read ஆகும் போது:

👉 Node.js next task execute பண்ணும்

இதுதான் Non-Blocking I/O.

⭐ Super Simple One Line

I/O = Data inside computer ↔ outside world communication

⭐

Final Interview Sentence

"Non-blocking I/O allows Node.js to execute multiple operations without waiting for each task to complete."
