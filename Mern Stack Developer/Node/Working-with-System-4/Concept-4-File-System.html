File System Module (fs)

Node.js-ல files மற்றும் folders handle பண்ண கற்றுக்கொள்வோம்

01
🌟 Step 1 – Real Life Question

Class start பண்ணும்போது கேள்:

"Server-ல ஒரு file save பண்ணணும் என்றால் எப்படி?"

Example situations:

🖼️ User upload image
📝 Logs store பண்ண
⚙️ Configuration file read பண்ண
👉 Answer Node.js-ல File System module (fs) use பண்ணுவோம்.
02
📋 Step 2 – Definition
👉 Board-ல எழுதணும் fs = File System module

Meaning: Node.js server-ல files மற்றும் folders-ஐ handle பண்ண உதவும் module.

Examples:

01

📄 file read

Server-ல உள்ள file-ஐ read பண்ணலாம்

02

✍️ file write

New file create அல்லது content write பண்ணலாம்

03

🗑️ file delete

Existing file-ஐ remove பண்ணலாம்

04

📁 folder create

New directory/folder create பண்ணலாம்

03
📦 Step 3 – Import fs Module

Students-க்கு first காட்டணும்:

index.js 
const fs = require("fs")
✅ fs என்பது built-in module
🚫 install செய்ய வேண்டாம்
04
⚙️ Step 4 – Important Methods
1️⃣ Read File

File content read பண்ண.

readFile.js 
const fs = require("fs")

fs.readFile("data.txt","utf8",(err,data)=>{
  if(err) throw err
  console.log(data)
})
Explain
→ readFile → file open
→ data → file content
file content console-ல print ஆகும்
2️⃣ Write File

New file create அல்லது content write.

writeFile.js 
fs.writeFile("data.txt","Hello Node.js",(err)=>{
  if(err) throw err
  console.log("File created")
})
Meaning
👉 data.txt file create ஆகும்.
3️⃣ Append File

Existing file-க்கு new content add.

appendFile.js 
fs.appendFile("data.txt","New content",(err)=>{
  if(err) throw err
})
4️⃣ Delete File
deleteFile.js 
fs.unlink("data.txt",(err)=>{
  if(err) throw err
  console.log("File deleted")
})
Meaning
👉 file remove ஆகும்.
5️⃣ Create Folder
createFolder.js 
fs.mkdir("testFolder",(err)=>{
  if(err) throw err
})
Meaning
👉 new folder create ஆகும்.
05
⚡ Step 5 – Sync vs Async (Important Concept)

Node.js-ல 2 types methods இருக்கும்.

Type Method Example Meaning Behavior
Async Recommended fs.readFile() Non-blocking Server wait பண்ணாது.
Sync Avoid fs.readFileSync() Blocking File read ஆகும் வரை server wait பண்ணும்.
Important Node.js-ல mostly Async methods use பண்ணுவோம்.

⚙️ Configuration File என்றால் என்ன?

Application settings save பண்ணும் file பற்றி கற்றுக்கொள்வோம்

1️⃣
Configuration File என்றால் என்ன?

Configuration file

Configuration file = application settings save பண்ணும் file

இதில் இருக்கும்:

  • 🔌 server port
  • 🗄️ database URL
  • 🔑 API keys
  • 🌍 environment settings

Example file:

config.json 
{
  "port": 5000,
  "db": "mongodb://localhost:27017/app"
}
Meaning
👉 application run ஆக தேவையான settings.
Node.js read
2️⃣
Node.js எப்படி read பண்ணும்?

Node.js இந்த file-ஐ fs module use பண்ணி read பண்ணும்.

Example:

readConfig.js 
const fs = require("fs")

fs.readFile("config.json","utf8",(err,data)=>{
  if(err) throw err
  console.log(data)
})
{ "port": 5000, "db": "mongodb://localhost:27017/app" }
Meaning
👉 program settings read பண்ணுது.
Real Example
3️⃣
Real Example

Imagine பண்ணு: server எந்த port-ல run ஆகணும்?

config file:

config.json 
{
  "port": 5000
}

Node.js code:

server.js 
const fs = require("fs")

fs.readFile("config.json","utf8",(err,data)=>{
  const config = JSON.parse(data)
  console.log(config.port)
})
5000
Result Server port value file-ல இருந்து வந்தது.
