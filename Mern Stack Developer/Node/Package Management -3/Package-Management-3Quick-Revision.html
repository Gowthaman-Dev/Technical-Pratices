NPM

Node Package Manager

Definition
NPM — Node Package Manager

Node.js packages install மற்றும் manage பண்ண பயன்படும் tool. Node.js install பண்ணும்போது NPM automatic-ஆ install ஆகும். External libraries install, dependencies manage, scripts run, packages publish பண்ண use பண்ணலாம்.

Play Store / App Store போல

Phone-ல் WhatsApp, Instagram install பண்றோம் — NPM-ல் express, axios install பண்றோம்.

npm install-ல் என்ன நடக்கும்?
1Package download + install ஆகும்
2node_modules folder create ஆகும்
3package.json update ஆகும்
5 Important Commands
1
npm install express
Express package install ஆகும். node_modules folder + package.json update ஆகும்.
local install
2
npm install nodemon --save-dev
Development time மட்டும் use பண்ணும் package. Production-ல் தேவையில்லை.
dev dependency nodemon eslint prettier
3
npm install -g nodemon
System / computer முழுவதும் install ஆகும். எந்த project-லயும் use பண்ணலாம்.
global install
4
npm uninstall express
Express package remove ஆகும். node_modules மற்றும் package.json-லயும் remove ஆகும்.
remove package
5
npm install
package.json-ல் இருக்கும் all packages install ஆகும். GitHub clone பண்ணும்போது use பண்ணுவோம்.
install all deps
node_modules Folder

Installed packages

npm install பண்ணும் packages எல்லாம் இங்கே store ஆகும்.

Dependencies of dependencies

Express install பண்ணினால், Express use பண்ணும் other libraries கூட install ஆகும்.

node_modules folder-ஐ GitHub-ல் push பண்ண வேண்டாம் — .gitignore-ல் add பண்ணுவோம். npm install run பண்ணினால் மீண்டும் create ஆகும்.

Most Asked Interview Q — Local vs Global vs Dev?

Local → Current project மட்டும் — npm install express
Global → System-wide, any project — npm install -g nodemon
Dev dependency → Dev time only, production-ல் தேவையில்லை — npm install nodemon --save-dev

Interview Sentence — Memorize

"NPM is the default package manager for Node.js used to install, manage, and update project dependencies."

package.json

Project Configuration File

Definition
package.json — Project Configuration File

Node.js project-இன் ID card. Project name, version, dependencies, scripts எல்லாம் இந்த ஒரு file-ல் store ஆகும். இல்லாமல் project manage பண்ண முடியாது.

Example File
{
  "name":        "mern-project",
  "version":     "1.0.0",
  "description": "Job portal backend",
  "main":        "server.js",
  "scripts": {
    "start": "node server.js",
    "dev":   "nodemon server.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.0"
  }
}

"name"
Project name
"version"
Project version
"description"
Project details
"main"
Entry file — server start ஆகும் file
"scripts"
Commands run பண்ண
"dependencies"
Required packages

100% Interview Q — npm install-ல் என்ன நடக்கும்?
1package.json-ல் இருக்கும் dependencies read ஆகும்
2npm registry-லிருந்து packages download ஆகும்
3node_modules folder-ல் install ஆகும்
4package-lock.json update ஆகும் (exact versions lock)
Final Combined Answer (NPM + package.json)

"NPM is the package manager for Node.js used to install and manage external libraries. The package.json file stores project metadata, dependencies, and scripts required to run the application."

Interview Sentence — Memorize

"package.json is a configuration file that stores project metadata, dependencies, and scripts for a Node.js application."
