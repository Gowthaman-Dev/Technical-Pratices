01

Node.js Basics

Tamil Explanation

Node.js என்பது JavaScript runtime environment. இதன் மூலம் JavaScript-ஐ browser இல்லாமல் server side-ல் run பண்ணலாம்.

Normally JavaScript run ஆகும் இடம்

  • Browser (Chrome, Firefox)

ஆனா Node.js வந்த பிறகு JavaScript backend-லும் run ஆக ஆரம்பிச்சது.

Node.js use பண்ணி நாம் செய்ய முடியும்

  • Web servers create பண்ண
  • REST APIs create பண்ண
  • Database connect பண்ண
  • File system operations
  • Real-time applications (chat apps)

Example (Simple Node Server)

server.js 
const http = require("http");

const server = http.createServer((req, res) => {
  res.write("Hello from Node.js");
  res.end();
});

server.listen(5000, () => {
  console.log("Server running on port 5000");
});

இதில்

  • http module use பண்ணி server create பண்ணுறோம்
  • Client request வந்தால் response send பண்ணுறோம்

Important Points (Interview) – Node.js features

Open source Cross platform Event-driven Non-blocking I/O Fast execution (V8 engine)
⭐ English Sentence (Interview)
"Node.js is a runtime environment that allows JavaScript to run on the server side using the V8 JavaScript engine."
02

Node.js Architecture (Single-Threaded Event-Driven)

Tamil Explanation

Node.js Single Thread + Event Driven Architecture use பண்ணும்.


இதன் meaning

  • One main thread மட்டும் இருக்கும்
  • Requests வந்தாலும் block ஆகாமல் handle பண்ணும்
  • Event loop use பண்ணி asynchronous operations manage பண்ணும்

Traditional Server (Multi Thread) — Example: Java / PHP

❌ Traditional (Multi Thread)

Client 1 → Thread 1
Client 2 → Thread 2
Client 3 → Thread 3

✅ Problems with Multi Thread

  • அதிக memory
  • slow performance

Node.js Architecture

Client Requests
↓
Event Queue
↓
Event Loop
↓
Non-Blocking Operations
↓
Response

இதனால்

  • thousands of requests handle பண்ணும்
  • memory usage குறையும்
  • scalable applications build பண்ணலாம்

Example

event-loop.js 
setTimeout(() => {
  console.log("Task finished")
}, 2000)

console.log("Next task running")
▶ Output
Next task running
Task finished

Node.js wait பண்ணாமல் next code run பண்ணும்.

⭐ English Sentence (Interview)
"Node.js uses a single-threaded event-driven architecture to handle multiple requests efficiently."
03

Non-Blocking I/O

Tamil Explanation

Node.js-ல I/O operations block ஆகாது.


I/O என்றால்

  • File read
  • Database query
  • API call
  • Network request

❌ Traditional languages-ல்

Read file → wait → next code run

✅ Node.js-ல்

Read file → continue next code → result later

இதுவே Non-Blocking I/O.

Example

non-blocking.js 
const fs = require("fs");

fs.readFile("data.txt", "utf8", (err, data) => {
  console.log(data);
});

console.log("File reading started");
▶ Output
File reading started
(file content later)

இதன் காரணம்

Node.js file read முடியும் வரை wait பண்ணாது.

Benefits

  • Fast performance
  • High scalability
  • Multiple users handle பண்ண முடியும்
⭐ English Sentence (Interview)
"Non-blocking I/O allows Node.js to handle multiple operations without waiting for each task to finish."
⭐
