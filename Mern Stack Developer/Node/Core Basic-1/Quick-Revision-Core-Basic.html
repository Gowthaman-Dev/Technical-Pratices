Simple Node Server — Code Example
server.js
const http = require("http");

const server = http.createServer((req, res) => {
  res.write("Hello from Node.js");
  res.end();
});

server.listen(5000, () => {
  console.log("Server running on port 5000");
});
const http = require("http")Node.js built-in http module import பண்ணுறோம்.
http.createServer()Server create பண்ணுறோம்.
res.write("Hello...")Client-க்கு response send பண்ணுறோம்.
server.listen(5000)Server port 5000-ல் run ஆகும்.

Interview Trick Q

Q: If Node.js has only one thread, how can it handle thousands of requests?
A: Using Event Loop and Non-Blocking I/O — Node.js doesn't wait for operations to complete, it processes them asynchronously through the event queue and event loop.

Non-Blocking

Non-Blocking I/O

I/O என்றால் என்ன?
I/O = Input / Output

Computer வெளியில் இருந்து data வாங்குவது (Input) மற்றும் data வெளியே அனுப்புவது (Output). Program external source-லிருந்து data read அல்லது write பண்ணும் operations = I/O.

File Read
fs.readFile()
file-லிருந்து data — Input
Database Query
User.find()
DB-லிருந்து data — I/O
API Call
axios.get()
external API data — I/O
Network Request
fetch()
network data travel — I/O

I/O = Data inside computer ↔ outside world communication

Blocking vs Non-Blocking
Blocking — Traditional (Java/PHP)
Read file → wait → next code
1. Read file start
2. Wait (server blocked)
3. Next operation
Server slow
Requests queue ஆகும்
Scalability குறையும்
Non-Blocking — Node.js
Read file → continue → result later
1. File read start
2. Next code immediately run
3. File ready → callback run
Fast performance
High scalability
Multiple users handle
Code Example
non-blocking.js
const fs = require("fs");

fs.readFile("data.txt", "utf8", (err, data) => {
  console.log(data);  // file ready ஆனதும் run ஆகும்
});

console.log("File reading started");
Output: File reading started (file content later)

fs.readFile()File read operation start — wait பண்ணாது.
(err, data) => {}File read complete ஆன பிறகு callback run ஆகும்.
console.log("File reading...")Node.js wait பண்ணாமல் next code execute பண்ணும்.
Execution Order — Step by Step
file read start ஆகும்
Node.js wait பண்ணாது — next code run
"File reading started" print ஆகும்
file read complete → callback run → content print

இதுதான் Non-Blocking I/O — Node.js முடியும் வரை wait பண்ணாது.

Interview Trick Q

Q: Why is Node.js faster than traditional servers?
A: Because it uses Non-Blocking I/O and Event Loop — Node.js doesn't wait for I/O operations, it processes other requests while waiting for I/O results.
