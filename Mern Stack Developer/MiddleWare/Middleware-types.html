1️⃣ Application Middleware
Idea: Whole application-க்கும் run ஆகும் middleware. அதாவது every request-க்கும் run ஆகும்.
app.use((req,res,next) => { console.log("Application middleware") next() }) // example: cors app.use(cors())
“Application middleware runs for every request in the entire Express application.”
🌐 every request → app.use() → next()
2️⃣ Route Middleware
Idea: Specific route-க்கு மட்டும் run ஆகும் middleware.
app.get("/dashboard", auth, (req,res) => { res.send("Dashboard") }) // auth middleware const auth = (req,res,next) => { // check token next() }
“Route middleware is applied only to specific routes.”
📌 /dashboard → auth middleware → route handler
3️⃣ Third-party Middleware
Idea: External packages (npm) use பண்ணுற middleware. Developers create பண்ணி npm-ல publish பண்ணிருப்பாங்க.
Examples: cors, multer, morgan, body-parser
import cors from "cors" app.use(cors())
“Third-party middleware are external packages installed from npm.”
4️⃣ Built-in Middleware
Idea: Express already provide பண்ணுற middleware. Examples: express.json(), express.urlencoded()
app.use(express.json()) // client send பண்ணுற JSON data read பண்ண
“Built-in middleware are provided by Express itself.”
🔥 “Express has different types of middleware such as application middleware, route middleware, third-party middleware, and built-in middleware. Application middleware runs for every request, route middleware runs for specific routes, third-party middleware are installed from npm like cors, and built-in middleware like express.json() is provided by Express.”
