🔐 Auth middleware · JWT · route protection

dashboard · token verify · 401 · interview prep
1️⃣ Problem (why we need this)
First interviewer-க்கு situation சொல்லு:
“Some routes like dashboard should be accessed only by logged-in users. So we use authentication middleware to verify the user token.”

Tamil idea: Login பண்ணாத user dashboard access பண்ணக்கூடாது.
2️⃣ Middleware code
const verifyUser = (req,res,next) => { const token = req.headers.authorization if(!token) { return res.status(401).json({msg:"No token"}) } next() }
3️⃣ Line by line explanation
const token = req.headers.authorization
Client send பண்ணுற JWT token read பண்ணுறோம். Example: Authorization: Bearer eyJhbGci...

if(!token)
Token இல்லனா → return res.status(401).json({msg:"No token"})
401 → Unauthorized, user login இல்ல.

next()
Token இருந்தா ➡ request next step-க்கு போகும் ➡ route handler run ஆகும்.
4️⃣ Route use
app.get("/dashboard", verifyUser, (req,res) => { res.send("Dashboard data") })
User /dashboard call பண்ணும்போது flow:
Client request → verifyUser middleware → Token check → Valid → dashboard route run
5️⃣ Flow diagram (interview super point)
Client Request ⬇️ verifyUser ⬇️ Token verify ⬇️ Valid → Route ⬇️ Response
6️⃣ Interview sentence
“In a MERN application, authentication middleware is used to protect routes. The middleware checks the JWT token in the request header. If the token is valid, it allows access to the route using next(); otherwise it returns a 401 unauthorized response.”
7️⃣ Super simple interview answer
👉 “This middleware checks whether the user has a valid token. If the token is missing, it returns an unauthorized error. If the token exists, the request proceeds to the dashboard route.”
