1️⃣ What is Middleware? (interview)
“Middleware is a function that runs between the request and the response in an Express application. It can modify the request, check data, perform authentication, logging, validation, or security before sending the response.”
Simple English: “Middleware is a function that executes before the request reaches the route handler.”

Tamil meaning: Client request → Route handler போகும் முன் execute ஆகும் function தான் middleware.
Client Request ➡️ Middleware ➡️ Route Handler ➡️ Response
2️⃣ Why use Middleware? (4 points)
🔐 Authentication user login check
📋 Logging track requests
✅ Validation data correct-ஆ?
🛡️ Security token / API key
🔐 Authentication: User login இருக்கானா check.
const auth = (req,res,next) => { if(req.headers.token){ next() } else { res.send("Unauthorized") } }
“Authentication middleware checks whether the user is logged in or authorized before accessing protected routes.”
📋 Logging: Server-க்கு வரும் request details track.
app.use((req,res,next) => { console.log(req.method, req.url) next() })
“Logging middleware records information about incoming requests such as HTTP method and URL.”
✅ Validation: User data correct-ஆ check.
if(!req.body.email){ return res.send("Email required") }
“Validation middleware ensures that the request data is correct before processing it.”
🛡️ Security: Token verify, API key check.
“Security middleware verifies tokens or API keys to protect the application.”
3️⃣ Middleware Flow & next()
Client Request⬇️Middleware⬇️Route Handler⬇️Response
“Middleware functions run sequentially before the route handler and must call next() to pass control to the next middleware.”
⚠️ next() important point: next() call பண்ணலனா request stop ஆகும்.
“The next() function is used to pass control to the next middleware.”
app.use((req,res,next) => { console.log("Middleware executed") next() })
4️⃣ Final Interview Answer (short)
“Middleware is used to execute logic before the route handler. It helps in authentication, logging, validation, and security. Middleware functions run between the request and response cycle and use next() to pass control to the next middleware.”
5️⃣ 📋 Logging Middleware – deep dive
1️⃣ Idea என்ன? Server-க்கு வரும் request details print பண்ணுறது. யார் எந்த URL call பண்ணாங்கன்னு server log பண்ணிக்க முடியும்.
app.use((req,res,next) => { console.log(req.method, req.url) next() })
Line by line:
• app.use() – Express-ல middleware run பண்ண use பண்ணுற method. “Run this function for every request”
• req.method – User என்ன request type use பண்ணான் (GET, POST, PUT, DELETE)
• req.url – User எந்த URL call பண்ணான் ( /users, /login )
• console.log – Server console-ல print ஆகும்: e.g. GET /users
• next() – Next middleware / route-க்கு request அனுப்பும். ⚡ next() இல்லனா request stop.
“Logging middleware prints request details such as method and URL in the server console to monitor incoming requests.”
