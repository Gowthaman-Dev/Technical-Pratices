dotenv & Environments

Development · Testing · Production — Tamil-ல் முழு விளக்கம்

1

Start with the Problem (Why needed?)

நம்ம project development, testing, production என மூன்று environment-ல் run ஆகும். ஒவ்வொன்றுக்கும் PORT, DB URL, API keys வெவ்வேறு இருக்கலாம். அதை code-ல hardcode பண்ணினால், mistake chance அதிகம், maintenance கடினம்.

Example:

❌ Bad practice app.js 
// Bad practice
const PORT = 5000; // dev
const MONGO_URL = "mongodb://localhost:27017/devdb";
இதை production-க்கு change பண்ண மறந்தால் app break ஆகலாம். ❌
ENGLISH SENTENCE FOR CLASS "Hardcoding environment-specific configuration can lead to errors when moving code between development, testing, and production."
2

Introduce .env Files

ஒவ்வொரு environment-க்கும் தனி .env file வைத்தால் easy. Dev-க்கு .env.development, Test-க்கு .env.test, Prod-க்கு .env.production.

Visual aid:

PROJECT STRUCTURE
project/ ├─ .env.development PORT=5000 MONGO_URL=mongodb://localhost:27017/devdb ├─ .env.test PORT=4000 MONGO_URL=mongodb://localhost:27017/testdb └─ .env.production PORT=80 MONGO_URL=mongodb://prod-db-server:27017/proddb
3

Show How dotenv Loads Config

Code Example:

app.js 
import dotenv from "dotenv";

// dynamically load based on NODE_ENV
dotenv.config({ path: `.env.${process.env.NODE_ENV}` });

console.log(process.env.PORT);
console.log(process.env.MONGO_URL);

Class Explanation:

  • NODE_ENV=development .env.development load
  • NODE_ENV=production .env.production load
  • process.env process.env-ல values access பண்ணலாம்
4

Practical Demo (Interactive)

  1. Create 3 .env files like above
  2. Change NODE_ENV in terminal:
    TERMINAL
    # For development
NODE_ENV=development node app.js

# For production
NODE_ENV=production node app.js
  3. Show output difference: PORT & DB URL change ஆகுது.
