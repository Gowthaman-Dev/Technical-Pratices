Environment Variables

Configuration Management — Tamil-ல் முழு விளக்கம்

1

Problem (Why we need configuration)

👉 "In real applications we have different environments like development, testing, and production. Each environment may need different configuration values."

Tamil idea:

App run பண்ணும் place change ஆனா values change ஆகும்.

Examples:

DEVELOPMENT

Local laptop

TESTING

Testing server

PRODUCTION

Live server

2

Example

.env 
PORT=5000
DB_URL=mongodb://localhost:27017/mydb
JWT_SECRET=abc123

Explanation:

KEY → DESCRIPTION
PORT → server run ஆகும் port
DB_URL → database connection
JWT_SECRET → token secret key
3

How we use it in code

Install dotenv

setup 
import dotenv from "dotenv"
dotenv.config()

Use value

app.js 
const PORT = process.env.PORT

app.listen(PORT, () => {
   console.log("Server running")
})
4

Why we use .env (Important interview point)

Advantages:

  • 🔐 Sensitive data hide பண்ணலாம்
  • ⚙️ Environment values change easy
  • 🚀 Production deploy easy
5

Interview Sentence

👉 INTERVIEW ANSWER

Environment variables are used to manage configuration values such as port numbers, database URLs, and secret keys. They allow different settings for development, testing, and production environments.

6

Super Short Answer (Interview)

👉 SHORT ANSWER

We use environment variables to store configuration values like PORT, database URL, and secret keys. This helps manage different environments such as development and production.
