dotenv என்றால் என்ன?

dotenv என்பது .env file-ல இருக்கும் secret values-ஐ Node.js application-க்கு load பண்ண use பண்ணுற package.

Example .env file 
PORT=5000
MONGO_URL=mongodb://localhost:27017/mydb
JWT_SECRET=mysecretkey

இந்த values-ஐ code-ல use பண்ணலாம்.

Usage 
console.log(process.env.PORT)
INTERVIEW SENTENCE

Dotenv is used to load environment variables from a .env file into process.env.

2

dotenv ஏன் use பண்ணுறோம்?

1️⃣ Security

Password / secret keys code-ல directly எழுதக் கூடாது.

❌ Bad practice
const password = "123456"
✅ Good practice
DB_PASSWORD=123456
process.env.DB_PASSWORD

2️⃣ Configuration manage பண்ண

Different environments இருக்கும்:

  • Development
  • Testing
  • Production

ஒவ்வொன்றுக்கும் values change பண்ணலாம்.

Example
PORT=5000

3️⃣ Code clean ஆக இருக்கும்

Database URL போன்ற values code-ல clutter ஆகாது.

Example
mongoose.connect(process.env.MONGO_URL)

Advanced dotenv Concepts

1️⃣ MULTIPLE .env FILES

Project-ல் development, testing, production environments உண்டு. அதற்கு ஒவ்வொரு environment-க்கும் தனி .env file வைத்தால் சுலபம். உதாரணம்: .env.development, .env.production.

We can use different .env files per environment to manage configuration safely.

dotenv.config({ path: ".env.production" });

2️⃣ DEFAULT VALUES

.env file-ல variable இல்லாத போது app crash ஆகாமல் default value கொடுக்கலாம்.

Provide fallback values if environment variables are missing.

const PORT = process.env.PORT || 3000;

3️⃣ TYPE SAFETY

process.env-ல உள்ள values எல்லாம் strings. Number, Boolean வேண்டுமானால் convert பண்ணணும்.

Convert environment variables to correct type when needed.

const port  = Number(process.env.PORT);
const debug = process.env.DEBUG === 'true';

4️⃣ SECURITY CAUTION

.env file-ஐ Git-ல் commit பண்ணாதீர்கள். .gitignore-ல add பண்ணணும்.

Never push .env to version control to keep secrets safe.

.gitignore
# ignore env secrets
.env
.env.*

5️⃣ DOTENV-EXPAND

ஒரு env variable-ஐ மற்றொரு variable-ல use பண்ணலாம். Example: BASE_URL=http://localhost:${PORT}

Allows referencing one env variable inside another.

PORT=5000
BASE_URL=http://localhost:${PORT}

6️⃣ DYNAMIC LOADING

Node environment (development/test/production) பறித்து அந்த environment-க்கு தேவையான .env file load பண்ணலாம்.

Load different .env files dynamically based on NODE_ENV.

dotenv.config({
  path: `.env.${process.env.NODE_ENV}`
});

7️⃣ CROSS-PLATFORM DIFFERENCES

Windows & Linux systems-ல் path or environment variable handling சிறிது வேறுபடும், கவனிக்கணும்.

Be aware of OS differences when using environment variables or paths.

Windows
set PORT=5000 && node app.js
Linux / Mac
PORT=5000 node app.js
