Server Creation

🖥️
"Express is a Node.js framework used to easily create servers and handle HTTP requests."
Client request வந்தா → handle பண்ணி → response அனுப்புவது = Server ✅
Basic Server Code
server.js 
import express from "express"

const app = express()           // Express app create

app.get("/", (req, res) => {
  res.send("Server is running") // Response send
})

app.listen(5000, () => {
  console.log("Server started on port 5000")
})

Request (Port 5000)
Express Server ← Handle பண்றது
Response

"Express is a Node.js framework used to easily create servers and handle HTTP requests. We create an Express app using express(), define routes using app.get(), and start the server using app.listen() on a specific port."

Basic Middleware Code
middleware.js 
import express from "express"

const app = express()

  console.log("Middleware executed")

  res.send("Users data")

Request Object (req) — 3 Types
req-types.js 
// 1️⃣ req.body → Form data
app.post("/register", (req, res) => {
  console.log(req.body)       // { name: "Gowtham", email: "..." }
})

// 2️⃣ req.params → URL parameter
app.get("/users/:id", (req, res) => {
  console.log(req.params.id)  // /users/5 → "5"
})

// 3️⃣ req.query → Query string
app.get("/search", (req, res) => {
  console.log(req.query.name) // /search?name=Gowtham → "Gowtham"
})
Propertyஎப்போ use பண்றது
req.bodyForm / POST data
req.paramsURL ல :id மாதிரி data
req.queryURL ல ?key=value data

⚠️ Error middleware ல exactly 4 parameters இருக்கணும் — இல்லன்னா Express error middleware-ஆ recognize பண்ணாது!
Parameterஎன்ன
errError object
reqRequest
resResponse
nextNext middleware
Route + Error Middleware Full Example
full-error.js 
app.get("/users", (req, res, next) => {
  try {
    throw new Error("Database error")   // 1️⃣ Error வருது
  } catch(err) {
    next(err)                           // 2️⃣ Error middleware கு அனுப்புது
  }
})

app.use((err, req, res, next) => {      // 3️⃣ Error handle பண்றது
  res.status(500).send(err.message)
})
1Route ல error வருது
2next(err) call பண்றது — Error middleware கு அனுப்புது
3Error middleware catch பண்றது — Response send பண்றது

"Express provides error-handling middleware to catch and manage application errors. It takes 4 parameters — err, req, res, and next. When an error occurs in a route, we call next(err) to pass it to the error middleware, which sends a proper response with the appropriate HTTP status code."
