Request & Response Objects

📮
1️⃣ Request & Response என்னன்னா?
Objectயாரு அனுப்புவாங்கஎன்ன
req (Request)Client → ServerForm data, URL params, query data
res (Response)Server → ClientReply / Result
"Request object contains data sent by the client, and the response object is used by the server to send data back to the client."

📬 Reply letter வருவதுResponse

Client (Browser)
Request (req)← Client data அனுப்புது
Express Server← Process பண்றது
Response (res)← Server reply அனுப்புது
Client (Browser)

5️⃣ Request Object (req) — 3 Types
req-types.js 
// 1️⃣ req.body → Form data
app.post("/register", (req, res) => {
  console.log(req.body)       // { name: "Gowtham", email: "..." }
})

// 2️⃣ req.params → URL parameter
app.get("/users/:id", (req, res) => {
  console.log(req.params.id)  // /users/5 → "5"
})

// 3️⃣ req.query → Query string
app.get("/search", (req, res) => {
  console.log(req.query.name) // /search?name=Gowtham → "Gowtham"
})
Propertyஎப்போ use பண்றது
req.bodyForm / POST data
req.paramsURL ல :id மாதிரி data
req.queryURL ல ?key=value data
6️⃣ Response Object (res) — 3 Methods
res-methods.js 
res.send("Hello")              // 📝 Text send
res.json({ name: "Gowtham" }) // 📦 JSON send
res.status(404).send("Not Found") // 🔢 Status code + message
Methodஎன்ன அனுப்புது
res.send()Plain text
res.json()JSON data
res.status()HTTP status code
7️⃣ Interview Answer (வாயால சொல்லு)

"In Express.js, the request object (req) contains data sent by the client such as form data, URL parameters, and query strings. The response object (res) is used by the server to send data back to the client using methods like res.send(), res.json(), and res.status()."

7️⃣ Interview Answer (வாயால சொல்லு)

"Static files are files like images, CSS, and JavaScript that are served directly to the client without any server-side processing. In Express.js, we use the express.static() middleware and pass the folder name to serve these files. For example, app.use(express.static('uploads')) will serve all files inside the uploads folder."

1️⃣ Error Handling என்னன்னா?
"Error handling is used to manage application errors and send proper responses."
Application ல error வந்தா → catch பண்ணி → proper response send பண்றது = Error Handling ✅

3️⃣ Error Flow

4️⃣ Error Middleware Code ⚠️
error-middleware.js 
app.use((err, req, res, next) => {
  res.status(500).send("Server Error")
})
Parameterஎன்ன
errError object
reqRequest
resResponse
nextNext middleware
⚠️ Error middleware ல exactly 4 parameters இருக்கணும் — இல்லன்னா Express error middleware-ஆ recognize பண்ணாது!
5️⃣ Route + Error Middleware Full Example
full-error.js 
app.get("/users", (req, res, next) => {
  try {
    throw new Error("Database error")   // 1️⃣ Error வருது
  } catch(err) {
    next(err)                           // 2️⃣ Error middleware கு அனுப்புது
  }
})

app.use((err, req, res, next) => {      // 3️⃣ Error handle பண்றது
  res.status(500).send(err.message)
})
Stepஎன்ன நடக்குது
1️⃣Route ல error வருது
2️⃣next(err) call பண்றது
3️⃣Error middleware catch பண்றது
6️⃣ Common HTTP Error Codes
status-codes.js 
res.status(404).send("Page Not Found")  // Route இல்லை
res.status(500).send("Server Error")    // Server problem
res.status(400).send("Bad Request")     // Invalid input

7️⃣ Interview Answer (வாயால சொல்லு)

"Express provides error-handling middleware to catch and manage application errors. It takes 4 parameters — err, req, res, and next. When an error occurs in a route, we call next(err) to pass it to the error middleware, which sends a proper response with the appropriate HTTP status code."
