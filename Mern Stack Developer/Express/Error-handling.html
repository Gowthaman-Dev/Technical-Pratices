1️⃣ “Website open பண்ணும்போது 'Server Error' அல்லது 'Page Not Found' வந்ததுண்டா?”

👉 Students answer: Yes
👉 Application ல error வந்தா server அதை handle பண்ணணும். அதற்கு use பண்ணுவது Error Handling Middleware.
2️⃣ Error Handling Definition (Tamil)
Error Handling என்றால் என்ன? Application run ஆகும்போது error வந்தால் அதை catch பண்ணி proper response send பண்ணுவது. Example errors: database connection error, API error, invalid request.
“Error handling is used to manage application errors and send proper responses.”
3️⃣ Error Flow (Board draw)
Client Request⬇️Route / Controller⬇️Error occurs⬇️Error Middleware⬇️Response
👉 Route ல error வந்தா error middleware handle பண்ணும்.
4️⃣ Express Error Middleware
app.use((err, req, res, next) => { res.status(500).send("Server Error") })
err error object
req request
res response
next next middleware
⚠️ Error middleware ல 4 parameters இருக்கணும்.
5️⃣ Example with Route
app.get("/users", (req, res, next) => { try { throw new Error("Database error") } catch(err) { next(err) } }) app.use((err, req, res, next) => { res.status(500).send(err.message) })
1️⃣ Route ல error create   2️⃣ next(err) call   3️⃣ Error middleware handle
